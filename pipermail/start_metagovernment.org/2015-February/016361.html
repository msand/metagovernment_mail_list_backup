you have to experience github to understand.
1. You just fork the data and do what ever you want.
2. You invite others to participate
3. You can ask for merging it back into the "official" data set, but you
have to get away from that way of thinking.

It's about democracy.
The repository WhiteHouse/2016-budget-data
<https://github.com/WhiteHouse/2016-budget-data> is theirs, but you can
have You/2016-budget-data

You may say, that is theory, but there is one very successful project out
there, called "nodejs". It's bringing javascript to the server side and its
trademark is owned by a company called joyent.
The way they stirred the project sucked and so most of the core developers
decided to fork it and bring it under an open governance model.
It's now called iojs and it's more successful than the original already,
because those programmers have high standing in the community and people
believe in them, so they make the switch.

They are still open to resolve issues with joyent and merge it back
eventually, but the whole point is, that - yes it's unpleasent in many ways
to use github, but awesome systems do not get build by single persons over
night - all this steady experimentation and moving back and forth is kind
of like an "open research" and once someone figured something out, people
start adopting it. This kind of model works particularly well in the
"javascript world" with a very liberal or anarchic approach towards having
an open source society in general.
Everyone can code whatever and just publish it for everyone easy to use -
and its just a command away, (e.g. "npm install You/2016-budget-data) and
then i start the project with (e.g. "npm start") and my browser opens
showing me what you worked on ....
I then change some parts, give it a nice message (e.g. "git commit "I added
more colors to your charts") and then push it back to you... or if you do
not like the terminal, you do that on github,

where you can comment on every line of code, PLEASE CHECK:
https://github.com/fairdemocracy/vilfredo-client/commit/bfc73baba1bf99228ceb376bb6ac03e310a92662#commitcomment-9656318
This is a comment on a project run by guys that also participate in the
context of metagovernment.org and i try to explain how github works.
