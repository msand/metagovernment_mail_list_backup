People search for AWESOME USER EXPERIENCE - it doesnt matter what your
"Matchism" is about if its not a nice interactive user interface! Because
everyone can produce tons of text which are TL;DR

If you want alpha testers or beta testers, then figure out who could be
potential "early birds" who need your software and are living on the
bleeding edge of tech, so that they would be willing to try it.... then
build a prototype (MVP = Minimum Viable Product) and get it out there. Host
it, ship it, .... etc..
There are facebook groups, linkedin groupd, twitter discussions, online
forums u can find through google, and lots of places.
Imagine who would be those early birds and go out there and search for them.

Create a so called "Landing Page" for your product, and PITCH it to them.
Make them subscribe to newsletter.
Create a Blog and inform them. In short => Build your audience.

So u feed and build your audience or community one step at a time and you
will have your users.
Put in "Google Analytic Script Tags" or use URL shortener like "bit.ly" to
be able to see what your users like and what not and adapt.

You PIVOT as long as necessary to get your testers and followers and grow
the movement.

GITHUB is just THE SOCIAL NETWORK for programmers and they constantly add
nice new features to make that collaboration more fun.
They have an API which is now heavily used by third parties to build
products around the GITHUB ECOSYSTEM :-)
Not that github is the answer, because in the long run, you would want
something more decentralised, but before you can decentralize, you need a
standard.
The standard is built using github - later you can build an open source
system that has the same "user experience"
