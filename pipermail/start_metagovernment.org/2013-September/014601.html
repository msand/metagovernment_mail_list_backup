Alexander, I agree that rules and manual steps will not work in the long
run. The difference here is that we want to be able to try something
quickly which usually means doing it manually until we are sure it works. I
often prototype in paper. Many lean start ups use many manual steps in the
initial phases until they have traction. I think putting a document
together would allow for better discussion in the decision on what we want
implemented before we start coding. If we have platform for prototyping
then we might be able to code prototypes fast enough to skip the manual
steps all together.
