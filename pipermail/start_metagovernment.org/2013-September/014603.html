The pirate party has a working solution and it is called "sync forum".
Maybe one could do a little research and find out about how they do it. I
bet they would be willing to share that tech.
Its not something i'm overly interested in, because i never use the forum,
but others do.

github is about individuals and collaboration.
javascript and nodejs is all about "npm" :-)
the awesome thing about javascript as a backend language is NPM :-)
publish your moduls to NPM or GITHUB and link those directly into your
project.
If you are able to split up the work into modules, then its totally awesome
to work together, because each nodejs project has a "package.json" file.
This "package.json" file lists all the external dependencies together with
the version number you are using.

So if you check your code into github and others check it out, all they
have to do after "git clone ..." is to run "npm install" and npm will get
all the dependencies with the right version number from npmjs.org or github
and put them into your local project folder... and then you are ready to go.


So what would be possible is, that we split work into modules and set up
some kind of dependency chain, then we all just configure our
"package.json" file and it will download all we need. everyone works on a
part of the project and because its possible to specify "versions", one
could already develop more features, but while they are not stable yet, the
others will just use the former version of the module.
