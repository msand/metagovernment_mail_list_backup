It's a good idea, but I should point out that so far this group has  
been conservative about editing the Main Page in particular. It is the  
first page most people see when encountering this project, and it  
should be something that broadly represents the consensus of the  
project.

While this project shares a lot of similarities with the wiki  
philosophy, it is a consensus governance group, not an open content  
group. In other words, we are more slow to act than a group of wiki  
editors. The difference is that something like Wikipedia is  
descriptive. it is easy to change an article and know that you are  
making it better. But if one person changes the Main Page, their  
vision is not necessarily the vision of the rest of the group.

In fact, we had about a month-long discussion just about the first  
couple of sentences on the current page, and another few weeks of  
reviewing the proposed new Main Page.

The way we have done the Main Page is to make a new one, such as:
http://metagovernment.org/wiki/Main_Page/2009-07
http://metagovernment.org/wiki/Main_Page/2008-10
http://metagovernment.org/wiki/Main_Page/2008-08
and then ask the list to review it. That way we are editing something  
that is not the front-end of the whole group, yet we are working as a  
group to improve the Main Page.

Your enthusiasm is greatly appreciated, and please don't mistake this  
for a rebuke. I am just pointing out that as a consensus group the  
Main Page in particular so far has been subject to a more conservative  
approach than a typical wiki page (same as the Main Page of Wikipedia,  
come to think of it).
