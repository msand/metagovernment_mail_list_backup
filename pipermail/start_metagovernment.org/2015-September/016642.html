I believe we need to find consensus about what metagovernment should be

The former has nothing to do with the latter.

1. Github has a "wiki" integrated, but even better, you can just "upload"
markdown files to the repository (or create a pull request to be accepted
to have a markdown file included)
2. Linking between markdown files will basically create a wiki.
3. We can give all the informationinsight into "the problematics".
4. Whenever a "change" is made or a PROPOSAL (= Pull Request) is created,
everyone is informed via email.

Github has way better tooling and is ideal for fast prototyping.
Once we know how to use the tools, we can extend and even move away from
github, but in the meantime, this allows much faster iteration.

Otherwise - if you could invite me to the metagovernment organization, that
would be great.
