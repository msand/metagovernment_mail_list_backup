Thomas von der Elbe wrote:
> Does our Metagov-wiki need to be a technically seperate wiki from
> the pollwiki? Because this is going to be there anyway I thought.

Technically we'd have no problem running a single wiki for the
combined content.  I would only hesitate if we required elaborate
extensions for Metagov's content, but apparently we don't:
http://metagovernment.org/wiki/Special:Version  The pollwiki is a
different matter: http://u.zelea.com/w/Special:Version

On the plus side (in favour of combining), it's easier to run a single
wiki.  They're somewhat labour intensive to maintain.

On the minus side, we'd probably put Metagov content under a
particular area of the pollwiki.  http://u.zelea.com/w/Category:Area
That would mean slightly longer URLs.  For example:

  http://metagovernment.org/wiki/Collaborative_governance
  http://metagovernment.org/wiki/M/Collaborative_governance

That way, the other areas (other "proto-pollwikis") could put their
own content on an equal footing with Metagov's.  For example:

  http://metagovernment.org/wiki/X/Collaborative_governance
