Hi Pietro,

I think the problems with trying to write the perfect question are one of
the reasons we need a discussion facility, in Vilfredo or in any other
decision-making process. And a proposal editing facility too. We need all
three elements running alongside one another: discussion, proposal writing
and editing, and voting.

Because the person writing the question is always going to have some kind of
presupposition - about what the question is about, or about the facts
relevant to the question, for example. So these can be pointed out in
comments, the question can be tweeked by other users, expanded into a bigger
question or split up into smaller ones, so that the users can see if they
agree on the relevant facts etc.

Then we can move into the voting stage, keeping the commenting and editing
process running at the same time so that people can say something like "Hey,
this is a really popular proposal but it doesn't resolve the issue we
proposed", and then modify the proposal or open up a new question which is
better defined.

It's true that people can influence the issue dramatically through the
discussion. I think that's good, as long as all the useful comments have a
chance to be heard. Any apparent consensus in the discussion still has to go
through the proof of the vote: perhaps there are silent dissenters out
there.

Cheers,
Paul
