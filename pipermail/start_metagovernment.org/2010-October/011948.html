Thanks Paul, good points. Vilfredo does not work well with parallel
question in the same question. So, reframing your suggestion, I would
suggest something like:
Question 1:
We would like to reach consensus on the meaning of various terms which
have been used in the context of collaborative governance. So please
define:
term 1

Question 2:
We would like to reach consensus on the meaning of various terms which
have been used in the context of collaborative governance. So please
define:
term 2

Question 3:
We would like to reach consensus on the meaning of various terms which
have been used in the context of collaborative governance. So please
define:
term 3

With each of those questions running independently.

And then once they all have reached a consensus we can decide what we
want to do.

Of course we can run a proto question where we ask: which questions
should we define. That is a good question, because people can easily
sum the results in the Pareto Front into a new proposal. Still I think
we should just let it free, and whoever feels a term needs to be
defined he can just run the relevant question. We could even advertise
those questions on twitter, so we get a result that does not represent
only us.

Pietro

> Here's another idea of how we might approach it (just putting this out
> there):
>
> We open a question in Vilfredo which says:
> "We would like to reach consensus on the meaning of various terms which have
> been used in the context of collaborative governance." Then we list the
> terms we want to define (with the possibility of additions) and invite
> everyone to define any or all of them. I'm sure we have some existing
> definitions on the wiki etc. to use as a starting point. These definitions
> can be refined and combined together into one answer as the question moves
> through the generations. If at the end we find that two or more terms have
> the same or very similar meaning, then we can try to find a method of
> choosing one over the other.
