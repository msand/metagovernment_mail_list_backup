Thomas would prefer a list that categorizes the various areas,
something like this: http://u.zelea.com/w/G/nav
Maybe we could add a Property:Jurisdiction_type (= global, local,

column in the table: http://u.zelea.com/w/Site:List_of_areas

> This is a proposal for improving area navigation in the central wiki.
> 
> Background
> ----------
> 
> We don't support multiple, independent pollwikis and streetwikis yet.
> Instead, just to get things started, we've divided a single, central
> wiki into multiple areas.  http://u.zelea.com/w/Category:Area
> This can cause navigation problems, especially if you want to go from
> one area to another.
> 
> Proposal
> --------
> 
>  a) Create a page called List_of_areas.  It lists the areas in
>     alphabetical order of display title.  Example:
> 
>         City of Toronto
>         Deutschland
>         Global
>         Piratenpartei Sachsen
>         Stadt Dresden
>         System area
> 
>     This is a technical page with no user-editable content, and no
>     implied relation among the areas.
> 
>  b) Set List_of_areas as the formal main page of the wiki, for those
>     technical cases (rare) that absolutely require it.
> 
>  c) Delete the existing main page (it serves no purpose).
>     http://u.zelea.com/w/Site:Main_page
> 
>  d) Rename "Main page" in left sidebar to "Area page".
> 
>  e) Disable "Area page" in the sidebar whenever the user is outside of
>     a particular area.  (Or at least grey it out, if it can't easily
>     be disabled.)
> 
>  f) Add "List of areas" (a) to the sidebar.
> 
> What do you think?
