Now we're talking design. My agenda is  this: add negative feedback loops
to the design of Money.

If you have a gripe with somebody you can tag that person. From that moment
on any 'coins' spent by this person will carry this tag. If your tag is 'I
hate this guy' nothing much will happen.

If your tag says 'criminal' you will be asked either to explain yourself or
risk being tagged yourself.

If I want to sell something my phone and the seller's phone will exchange
coins. My phone will scan each coin and reject based on a pre-configured
number of tags. If we need to exhange 500 coins and one coin is rejected
that person will need to have 501 coins to close the transaction.

This design - if ever accepted - would alter the work beyond recognition.
