On 8/25/2013 8:33 AM, Alexander Praetorius wrote:
Browser Plugins-Extensions do exist, so its enough to structure the email by
using html and put it inside of a container and give it an ID, then use that
browser plugin to detect that container and add css+javascript as needed to
the DOM in order to make it interactive.

With stand alone email clients its not that easy, but some of them (ex.
outlook) do support plugins too, and i guess thunderbird does so too.

Sadly, i'm not so sure about mobile email clients.

An alternative could be to provide only information and a button that says:
"REPLY", "REPLY ALL", "FORK TOPIC", .... and whatever else we need.
Those links could be "individualised" by the mail server and serve as
temporary password, like the links you get when u requrest a "password
reset" service when you have forgotten your username/password.
That would open up the browser with an "email client" that has all the
functionality we need for our proposal/discussion/proposal/... tool and
might be used to comment, highlight, fork, and structure my response to a
proposal in those ways i want, then "send it" back to the mailinglist.

Using that method, people would still receive emails the normal way, where
the whole email might look like our browser based interactive email client
(that supports javascript) and has buttons for "REPLY, REPLY ALL, and so
on..." which are just links which open a browser, loading that email content
and serve as an alternative "email client" then.
