>> How about a prototype website instead of using email or plugins. I think

>> requirement list like:
>>
>>     * A User can enter an issue
>>     * Users can create proposals to solve the issue
>>     * Users can create objections to the proposals
>>     * Objections are voted on? (I'm a little lost here)
>>
>> I would prefer a system that has no voting at all. I have a prototype of
>> a way to decide if a proposal should succeed that does not use voting.
>> Instead of voting people add reasons for or against passage of the
>> proposal. Each statement for or against can also be affected by statements
>> for or against it in a hierarchy. An objection could be made against the
>> proposal but then other people could object to the objection, for example,
>> by adding a statement that the objection is a logical fallacy. The system
>> weighs all the pros and cons and displays the confidence in the proposal
>> (and all the objections). I feel this is an example of a way to handle
>> objections without voting.
