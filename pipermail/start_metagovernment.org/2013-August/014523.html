Gmail does not allow JavaScript to run (
http://superuser.com/questions/430562).

I have been developing software (including email based software) for over
20 years and I can not think of a way to prototype this consensus building
application in just HTML and CSS without links to a website.

I propose that we create the application as a website and use email to
provide detailed of information to the participants and provide a link for
them to open up the interactive functionality on a web page.

We are not proposing to use a website to replace this email list.

Alexander, I believe that the only option to convince us to use email as an
interactive medium is  to either:

   - Find a developer that can implement interactive functionality in HTML
   and CSS without JavaScript
   - Propose a way to allow the functionality to work without JavaScript or
   websites.

You can see this debated at http://allangles.com/14
