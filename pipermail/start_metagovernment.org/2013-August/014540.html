No, there is no waterfall model, where you do a prototype in isolation and
if its sufficient, you start producing the "real mackoy".

Lean Startups start with a prototype which is used by customers as soon as
possible and they grow the prototype with their customers.
Just implement a single button in emails which does something that raw
emails could not do before, then use it and get feedback (we could use it
here), then we add the next button or manipulate the existing one... that
way, we have an interating process
(implement a feature, use it in our discussion, improve it or implement
another feature)

All this could be hosted on github.com to be downloaded by any person to
use that functionality too. if that happens often, we could start reaching
out to them and getting their feedback too. That way we avoid "ivory tower
stuff" (where ivory tower stands for: "lacking usability")
