@About MIRROR VOTING
Thats another one. Who should decide about the rules for mirror voting?
Sure, there should be experts that should give their input for the
aspect of the software that will support the "protokoll" of mirror
voting between 2 micro communities, but it should be the communities
that negotiate how mirror voting takes place.
Maybe experts could decide on a mechanisms between every 2 software
tools that implement a different decision finding process and how the
results map to each other...
... but my guess is that many assumptions have to be made and you
cannot be sure if a micro community that uses a tool will agree to
these decisions, so they should be allowed to negotiate their own
translation of votes to other systems with the other community that
uses a different tool.
