> Hi Jonas,
>
>> I'm building http://gov.e2d.se/ on top of an application framework I
>> developed, inspired by RDF and the semantic web. It's uses a realy
>> powerful and flexible nodal network database.
>
> Can you describe more about that?  I'm especially interested in how
> the network aspect fits with voting or decision functions.

There is a built-in history of all the links in the network. You can for 
example query the state at a specific point in time. This is useful for 
displaying the participation percentage for a voting in a closed group. 
You can count the number of members in the group at the time of the 
conclusion of the voting period.

There are built in tools for simple inference in the form of syllogisms 
that are used for the class hierarchy and can be used for things like 
cascading delegations.

But It will be of most use for more general connections between 
different versions of proposals, with connections to supporting facts 
and more.
