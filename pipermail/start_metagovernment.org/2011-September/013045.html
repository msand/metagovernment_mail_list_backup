I dont have a specification. But i don't think its worth the effort of
creating one.
The intention of "Simple-to-use-tools" was to make the installation and
registration process easy, so someone who knows nothing about computers but
how to open a webbrowser, clicking on download links and installing a
program on a windows machine and maybe knows how to use gmail or outlook
could manage to host a server for his friends or his community.
Would be nice if he could also do it by only using an android phone.
...the best option would be to make every metagovernment-tool available in
such a way. What is a "simple-to-use-tool" when it comes to the actual
consensus process within a micro community is up to them. metagovernment
tools will compete and hopefully can be given by the micro communities
through services like "uservoice.com" and the like.

What i mean by this "tool-builtin-broadcasting-updates-functionality ", is a
mechanism like firefox offers it.
If i write a plugin for firefox, i could upload it to mozilla and i could
release new versions and firefox will inform users and autoupdate my
plugins.
There might be similar mechanisms in other browsers. Which platforms should
be supported? Which of the several options would be the best?
I think programers have to give there input on this, because they know
better of the options that exist and they could estimate the effort it would
take to use one or several of them.
It also might depend on the experience or if programers are familiar with
one of the mechanisms...

All i wanted to say is, it is a good thing to have an infrastructure that
could deploy metagovernment tools even if they are only in alpha or prealpha
phase, because if someone decides to download the installer, chooses a tool
and gets all the autoupdates through mechanisms like the one that is
supported through firefox and which lets you, as a programmer, autoupdate
your plugin on every system that has installed it. ...this would make it
possible for prealpha users to give feedback and what they like and what
they dont like.

It creates an option for people who are interested in metagovernment tools
to play around and grasp what the current state of the technology is...

We don't have to code everything on our own. I'm not into it, but APACHE
MAVEN comes to my mind when i hear of JAVA. I think a lot of code of at
least some of metagovernment infrastructure and tools is written in java,
but probably MAVEN supports other languages as well and probably MAVEN could
help to incorporate services, like the autoupdate or autodeploy of mozillas
firefox and in general MAVEN is a nice tool when it comes to deploying one's
code.
