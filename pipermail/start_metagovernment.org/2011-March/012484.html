Sorry, yes of course not through a prediction engine. I was not aware, 
that the paragraph was limited to this. There must have been an older 
much broader version of this text.

BTW, I know, that there is much skepticism out there in the 
e-dem-community concerning this type of delegation. They feel, that 
someone (X) who has no clue on the subject, but votes for it, because A 
votes for B votes for C which votes for it, shouldn't be allowed. 
Instead just C should be allowed. An idea: How if we let each user 
decide, how to filter the results? So one could only count C.Or one 
counts all equally. Or one could decrease the voting power the longer 
the chain: C has 1 vote. B has 0.5 votes. X has 0.1 votes. ?
