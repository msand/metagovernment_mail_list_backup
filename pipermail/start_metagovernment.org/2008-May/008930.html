> My initial vision was a single distributed behemoth, like Wikipedia.
> Have a look here for their architecture:
> http://meta.wikimedia.org/wiki/Image:Wikimedia_Servers.svg
> http://meta.wikimedia.org/wiki/Image:Wikimedia-servers-2006-05-09.png

> As others noted, that makes the system more vulnerable to directed
> attack and makes it less flexible (which results in forks anyway). I
> can certainly see the benefit of distributed implementations. So I
> wold like to propose we work toward a synthesis, of course.
>
> Maybe we need to envision Metascore as something like a software
> standard (such as UNIX) and also as something like a communication
> standard (like Internet Protocol).
>

Oh no no no. It has far too complicated a use to be a standard with
different implementations.


> As a software standard, it would be a source code which is published
> in one form, but which is expected to be modified in its adaptation to
> any particular community. Hopefully the default configuration will
> work for most communities, but people can adapt it or fork different
> projects as they please. They can even use completely different
> software if they choose.
>

We can make it generic enough to be used by anything, so why expect
communities to need programming know how to be set up?


> As a communication standard, it would enable every community to
> interact with every other community, regardless of how they have
> tweaked their software. An individual would have to set up an account
> on any community they wished to participate in, but most of their
> preferences would be carried over from their personal settings. and
> communities could "borrow" scores and laws from each other, allowing
> "levels" of government to form themselves.


Distributed revision control is a wonderful thing. It would solve this
trivially (almost).

Just have a piece of software that uses a DRCS as a backend and under that
saves everything as plaintext. Make sure that people are able to add their
voice using but git and a text editor, but make available a client that lets
one browse through a graphical representation of everything and edit in a
custom-designed editor etc'. Make the protocol simple enough that it's easy
to taylor tools around it, but make one such tool the official "do
everything" tool because otherwise bugs in different tools will make it
HELL.

2c.
