If we're talking about something that governs the development of the
application, then please feel free to challenge the capabilities of the
participant by requiring the installation of additional software.

However, if we're talking about an interface design document, then we should
probably use something more "user friendly" such as a Wiki or a Google
Shared Document. This would allow for the greatest number of participants,
assuming the controls are as simple to use as a word doc.

I'm not saying we're all basic users. We're just not all programmers. :)
