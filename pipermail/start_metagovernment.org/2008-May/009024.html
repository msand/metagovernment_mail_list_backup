Back in the day, when command lines were replaced by windows, UI was exactly
that: User Interfaces. These days, the industry prefers the term "User
eXperience" to describe the environment in which the application is being
used. My point isn't to correct anyone from typing "UI," as the term still
applies to all the buttons and windows on desktop; but, for our discussion,
I believe it is just as important to consider the "user environment" as it
is to discuss the code.

The first thing that comes to mind is the necessity of both mobile and
public computing (which includes ATMs & kiosks as well as library
terminals). This means that iPod interfaces aren't secondary considerations,
but just as critical to include everyone without a personal laptop. For
example, the medical application I'm currently working on is primarily meant
to run in a cross-platform browser. However, as the product must also be
accessible through a mobile device, I designed a highly "modular" UI that
displays a certain number of "blocks" depending on the size of the screen.

So my question is: how do we envision Metascore as an aspect of people's
lives? Is this a Microsoft Visual Studio meant for "hardcore legal code
developers," a social network for people to connect with others in their
community, or an account manager that provides access to all the laws and
finances in their district? With all the talk of Wiki it sounds as though
we're designing an enormous law book that people can use, in part or whole,
in their own community.

> Wiki systems have proven that a discussion can reasonably be held
> within a text file under version control.
>
> The alternative would be to hold the proposals for legislation in a
> VCS and the discussion and ratings in a database.
>
> However, Metascore relies on the distributed nature of a DVCS to
> easily coordinate between many versions of the data in a decentralized
> way. Thus, it is essential that the files storing the discussion and
> ratings are kept within the DVCS repository. That being a given, there
> are a few advantages to text over a binary database format:
>
> - Plaintext plays better with the DVCS, especially:
>  * Delta storage means much smaller footprint
>  * Merging works (working with files that can't be merged and change
>   often, in a DVCS, is a nightmare, which means we would otherwise have
>   had to write a tool to merge databases)
> - Generic tools that exist on every developer's computer, specifically
>  a DVCS and a text editor, are sufficient to read and manipulate the
>  entire data model, thus making the data independent of
>  Metascore. This has many advantages, like being able to debug, to
>  validate the data and to participate in a discussion from a computer
>  where you cannot install Metascore.
> - It is much easier for the power user to automate tasks and for the
>  developer to create scripts and tools that manipulate the data.
>
> In general, it helps make the data independent from Metascore, which
> is a 'very' good thing.
