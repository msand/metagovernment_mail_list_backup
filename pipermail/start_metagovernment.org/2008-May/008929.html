The issue of transparency is solved if everything is stored in a plaintext
directory structure under distributed revision control, because then every
user has ALL the data about all the actions that ever happened without
needing to keep separate "logs".

The issue of user security can be solved with crypto-signing everything with
a government issued key that is a physical apparatus (a small USB key like
the eToken made by Aladdin Knowledge Systems that I used to work with,
perhaps a version that checks user biometrics before it agrees to be
activated to make stealing harder).
