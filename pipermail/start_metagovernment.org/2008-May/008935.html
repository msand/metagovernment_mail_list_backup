This might be a little out of left field, but I'm thinking about this
possibility of setting up universal user accounts that are shared
across all implementations / installations.. to, at the very least,
give the sysadmin the option of using them. Forgive my ignorance here
with the lingo, but I am thinking of something like domain name
servers, where all servers reference each other (perhaps like mysql
replication) and changes propagate among them, so that any server can
add a user account and all servers can reference user accounts created
on other servers.

My thought is, one install is the 'primary' for any given account, but
the user can reference that account from other installs by entering
their username/password combination -- once referenced, the user uses
the 'same' account on both installs though he/she authenticates for
each separately (to ensure no security loopholes). The fact that the
account is 'shared' means that there is the same authentication
process (eg, password) for all installs. User activity and rating can
be shared/merged between installs. The user can choose whether, in
referencing his account on the 'primary', to allow modifications to
the account to be made on only the master install or on other
installs.

Thoughts on something like this?

Is there no use in allowing user accounts be shared between installs?
Or, do we want to leave user accounts/authentication open for the sys
admin to configure using third-party apps, etc?
