Mike, I think, whats wrong with the overplan approach is the same thing
that is wrong with a waterfall project management.

1. Waterfall project management is bullshit, but in theory it makes sense,
only in practice it doesnt, because the actual PROCESS of how things get
done in reality is different.
All the phases of project management happen in parallel or if you want...
all the phases are re-iterated again and again. So i think the approach is
wrong.

2. For what to put on the metagovernment.org website, ...we blanked it and
now try to collaboratively make up the content.
What about those "overplan process", what about u suggest one element of
the process and then the next and we find out what this process should look
like through discussing it instead of you doing upfront everything :-)

Because you have thought about it already, you probably could easily
explain all the phases and why they are necessary or what would happen if
we for example would drop "Underplanning" and would directly jump to phase
3?
