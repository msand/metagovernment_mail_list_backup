Hi Scott, hi Steve, hi everybody.

Vilfredo is available and ready to use at http://vilfredo.org and I DID
propose to use Vilfredo for the discussion. It is in the discussion
archives:

http://metagovernment.org/pipermail/start_metagovernment.org/2014-May/006535.html

My message did not receive any attention to the point that I wondered if it
was even received by the list (now I checked and it was there).

Vilfredo could be used very well to decide the homepage.

Right now I am working very hard (as in every morning at 8am I have a skype
call) to develop the next Vilfredo. Vilfredo 2.0. But the original Vilfredo
is at http://vilfredo.org and works perfectly well.

For the records this was my proposal:

Hi,
speaking strictly for Vilfredo, but with a general view to the other tools.
Each tool is different and it is different the kind of questions and the
contexts in which it works well (or at all).

Vilfredo works under those conditions:
-5-25 people
-a situation where people are all willing to read each proposal (in a
future version we will try to change this but for now it is a necessity)
-a question where we are looking for a single answer.

The last point requires a bit of explenation: if we are saying what shall
we do for the US economy (or the Italian Economy), well there are plenty of
things that need to be done. And Vilfredo is not the right tool because it
permit a proposal to dominate another one, even if they have nothing to do
with each other.

Instead if you ask: how shall we build this car, where shall we go to eat
this evening, what shall we write to the MG homepage, then Vilfredo works
really well, because you are looking for a single answer. And if there are
multiple answers, you need to try to integrate them, or you will ignore one
instead of the other.

So saying that we shall use Vilfredo for each MG decision for 6 months does
not make sense. Vilfredo should be used only for certain metagovernment
decisions.

For example we could use it now to decide the homepage.

In this way, one week to propose what should be written,
one week to vote.
If we reach consensus we write what is in the consensus.
If we do not reach consensus, we give the alternative versions, of all the
proposals in the Pareto Front.
In the meantime, in the background we keep on running Vilfredo as a slow
buring, while trying to find a consensus.

Similarly Loomia works better for simpler decisions with many more people.
I think Loomia and Vilfredo could integrate very well. In fact you should
start with Loomia and once someone asks to use Vilfredo, provided there are
not more than 25 people, a Vilfredo question starts instead.

And so on.
