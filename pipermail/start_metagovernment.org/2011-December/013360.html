> Ok, but since it is all about politics and sometimes about really important

We can expect both the number of proposals, the number of clusters of
proposals to grow as the number of participants grow.

I think this is one of the cases where we need to work on bigger and
bigger systems, and it is unfeasible to evaluate now how the system
will behave when we have millions of people.

I was presenting three tools:
(1) a tool to aggregate proposals in clusters
(2) a tool to let some people in a cluster clarify it
(3) a tool to integrate different clusters.

Now it is possible that the number of clusters grow slower than the
number of proposals, with the effect that the second tool does not
scale up. This might be balanced by making smaller clusters, for
example. Or having multiple groups work in parallel for each cluster.

In short I think those tools can push the system further on
scalabilty. Maybe not all the way yet. But it might be good enough for
now.

Yes and no. Yes it's politics. But there might be something deeper.
I can only wave hands about it, so please forgive me. Thinking aloud:
Once you aggregate groups of people, the way in which you aggregate
you are essentially hiding information (how people disagree, who,
why). This is quite inevitable. But the way in which you do it can
have serious effects in the way those groups can then move to the next
level.

The fact is that setting up those fractal system is an old solution.
It has been used pretty much in any political system in the world. But
even when they tried to avoid them to become a source of centralized
power (like in communist Russia) it did not work.

A group of people discussing works fine. A group of group of people
often leads to bad decisions. This is why in the system presented in
the video have a final poll leading back to the people.
