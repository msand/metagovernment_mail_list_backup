I think this is an excellent idea!
As Chris Cargile mentioned, we've pontificated the generalities, but the
need for a system which enables us to discuss and rate topics is becoming
quite clear.

It would be wise to look at the current Alpha and determine the steps
necessary to import the Efficasync documentation. This would not only
provide a clear path of development, but allow us to test the software with
a particular goal in mind, while giving everyone the ability to participate
in the voting process.

I'm still trying to set up the development environment on my end in hopes of
templating the process. I'll be looking at the current functionality and the
essentials to the requests in hopes of finding the path of least resistance.
