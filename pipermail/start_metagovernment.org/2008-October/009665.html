Hi Darwin,

Darwin O Connor wrote:
> Scott Lewis wrote:
>> For example, in my own experience in the open source software world
>> (Eclipse), reputation plays a very important/critical role in the
>> developer community, but it's still sort of a mystery to me how
>> reputation among the technical community is/should actually be
>> supported.  The technology for EF doesn't explicitly support reputation,
>> but it's still used for reputation (of course).   Actually I think
>> people would use almost anything for reputation...because it's such a
>> meaningful thing to many people.
>
> That's another problem I have with a reputation system. People would 
> treat it like a game, like they did on slashdot 
> <http://slashdot.org/faq/com-mod.shtml#cm706>, and do thing that they 
> think would improve their reputations rather then voting in what the 
> believe in.

Slashdot's reputation system is not necessarily a good one...so using it 
as an example of the 'badness' of reputation systems in general doesn't 
really make sense.

But as for the general point about the undesired aspects of a reputation 
system...For argument I would probably go so far as to assert that if 
you put together a set of people that regularly interact (via network 
and/or other means) that you would eventually get some form of a 
reputation system...whether you intended to or not, or whether it's 
explicitly supported by technology or not.  So I can't really see the 
harm in encouraging reputation based upon something valuable (like 
community consensus and open review) rather than by some other, more 
traditional means.
