Very interesting. It has obviously evolved a lot respect to how it was
when we first spoke about it.

I would suggest that you try to pull the information from one page
from Vilfredo. We now have generational page, so that each generation
has all the information about that generation (proposal from before,
new proposals, by whom, who voted what, resulting pareto front, graph
of the voting structure, key people, and so on) all in one single
page. You can access it from the history page of a question. We still
lack a sensible way to move between the pages. It's on the todo list.

so that one page should be the right page for you to get all the info.
Or the history page. I think the history page of each question should
have its own rss, so the information can be read in machine readable
format.

Regarding Pareto Efficiency in theory and in practice, I don't know. I
would have to study that in the details. Beside we are moving on from

>Can your users draft their
> answers in wiki pages?

Of course people are free to do what they want, but so far it is not
implemented inside VgtA in any way shape or form. People would have to
write in a wiki page that they want the proposal and try to invite the
people there. We have an example of this happening in a recent
question:
http://vilfredo.org/viewquestion.php?q=113

We have thought about, when you have two key people of the same
proposal, to invite them to write collaboratively the answer. We
haven't implemented yet, but is in the "to think over" list.

But all this has to do with people working on editing an existing
proposal. Not so much in writing a proposal from scratch. Maybe we
could have a place where all the people that have been invited to a
question are listed, and then you can invite some to write a
collaborative text. The problem in all this is that it undermines the
whole structure of Vilfredo. No big deal with that, IF that works, but
our experience is that wiki by themselves lead to wikiwar, so vilfredo
was trying to generate an alternative.

An idea I had from a long time is to set up a Vilfredo system inside a
wiki. And this maybe could be an integration of Votorola with
Vilfredo. The idea is that when there is a wikiwar, then (and only
then) Vilfredo would be used. And only for the paragraphs that people
disagree with. Each party writing alternative versions, each person
voting all the versions that support, and then the Pareto Front would
all be represented in the wiki, side by side. So that no one solution
is above the other.

I think this could be used, for example, in the IPCC (in
http://IPCC.ch) to write their report, and would generate a better
result. You know that for something to appear in the IPCC report it
has to be supported by a certain percentage of scientist. This makes
the report ALWAYS err on the safe side (which is dangerous). Always
claiming that the clime will be less bad as it turns out to be. Giving
the possibility to have the various possibilities on the side would
prevent that.
