Why is it not ready for an open alpha?

I don't know the 4 alpha users that advised us that we're not ready for
beta, but I suspect its mainly for usability issues, that's what I think.

It's already possible to use votorola and we did when we wanted concensus
about the websites redesign, didn't we?
People can create accounts and they could start using it. I admit, at the
current moment it's very hard for people to use it, because it can be very
confusing and they will often wonder where to click or how it is supposed to
work. I think the sooner we get feedback from users, the better it is.

I doubt it will be a good thing if 3 or 4 months of coding will go into the
project without feedback from users.
In the end, a lot of code might end up in the waste bin because there will
always be many ways to put a concept into reality.
The goal is to make users familiar with the concept so that they can grasp
and start using it and I believe this only works when they are part of the
process.
Some Joe Average will give feedback that will influence the coding in a way
that makes it usable for all Joe Average's similar to the one who gave the
feedback.
The more different types of Joe Average we have, the more valuable the
feedback becomes, because it makes the system usable for every group of
users that is similar to one of the Joe Average's that gave us feedback :-)

I think if facebook would have started with all the features it has today it
would have failed. Every feature needed its time and had to be introduced
slowly. One could even say socially engineered into the userbase or
something like that. Features where introduced carefully one at the time
until enough users accepted it. There is still a lot of discussion and
outcry when facebook changes anything. It's the same with "semantic web".
There are many ivory tower solutions which work great as long as they stay
ivory tower, but as soon as you want it to be adopted by the mainstream it
always fails, because it's too complex to put it into practice, therefore
companies have put a lot of thought into it and a current attempt for a next
step in making semnatic web becoming mainstream reality is "schema.org".

Like I've mentioned earlier, there are a lot of services like
(uservoice.com) and many others that could be used by us to keep track of
all the features that testers would want. I think there are enough people on
this mailing list who would offer to administrate a service like
uservoice.com. If no one would, I would do it.

I think, like others have said earlier on this list, that I charge open
doors. Sure, I'm not so much into the current state of the tools and
therefore might miss essential points, but I know it's already possible to
create accounts and to use the tools as we did in the past.
So it's mainly about the right way of doing things or the best next steps to
take and I think the earlier we make it possible for anyone who's interested
in the tools to use them and provide a nice & easy way of giving us
feedback, the faster we will succeed.

An important feature of metagovernment tools should be a mechanism to give
feedback on anything, but until then it might be a good idea to use one of
the tools which are out there already.
