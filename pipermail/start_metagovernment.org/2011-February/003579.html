I wrote the below about a month ago, and received several affirmatives  
and no opposition. If there is still no opposition by this weekend, I  
propose we switch the Metagov wiki's copyright to Creative Commons  
Attribution-ShareAlike:
http://creativecommons.org/licenses/by-sa/3.0/
Note that this is the copyright Wikipedia uses:
http://en.wikipedia.org/wiki/Wikipedia:Copyrights

If you would like to view other CC licenses, here is their overview:
http://creativecommons.org/licenses/

Any objections or counter-proposals?


On Jan 3, 2011, at 7:16 PM, Ed Pastore wrote:

> This discussion started a couple of years ago and never resolved. In  
> part maybe because it is so boring...
>
> The issue is the restrictiveness of the current copyright on the wiki:
> http://www.metagovernment.org/wiki/Metagovernment:Copyrights
>
> It seems innocuous enough, but the "non-commercial" stipulation  
> makes us "closed" in the eyes of the open content community. One  
> important ramification of this is that we cannot host the wiki on  
> the TuxFamily.org site because they only support completely free/ 
> open projects.
>
> So there are two questions: should we remove the NC, and if so, how?
>
> I asked the Open Knowledge Foundation about the second question
> http://lists.okfn.org/pipermail/okfn-discuss/2011-January/007529.html
> and the one response I got so far thought one of my ideas was  
> plausible: to ask this list if anyone objects to the port, and if  
> nobody objects after a long period (say a month) to then change the  
> copyright.
>
> If we can resolve this, we can take a big step toward releasing this  
> project from my sole control and opening it up to other  
> administrators.
