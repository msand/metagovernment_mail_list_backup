Time for me to speak up again. As always, I agree with the majority of your
points Scott. As before though the proxy based mechanism of decision making
creates more problems than it solves I think.

I will define what I believe your proxy system to be, correct me where I'm
wrong.

- A new user to the system would choose multiple proxies from different
industries to represent their vote because they don't have time to read
every proposal.

- The new user would plug in a number of their core beliefs into the system
and then the system would recommend proxy users to attach to based on their
past voting habits. i.e. one proxy for agriculture issues, one proxy for
transportation issues.

- These proxies would only control the new user's vote in industries that
the new user agreed to.

Here is the first problem, what happens when multiple proxies intersect?
Many proposals will surely encompass more than one industry at once, our
new user will have two or more proxies voting different ways. Will it then
be up to the new user to read all the different reasons for each proxy vote
and determine which one best represents them?

- Once the proxies vote the user will be able to change their vote if they
don't agree with their proxy.

A bunch of questions here. What if their proxy doesn't vote, are they allow
to abstain? Will there be an alert that they haven't cast a vote? Will all
the users then have to read the whole proposal in the reduced time limit
now? What if you've got a proxy who's got a proxy? You agree with the
initial proxy but then your proxy changes their vote which in turn changes
yours. Do you still have time to change yours back to the initial vote? How
long is the time period? This confirmation of votes could go on for way too
long.

- Popular users will no doubt emerge and be in control of thousand of
votes, essentially deciding results themselves.

Is that not too much power for an individual user? Will these users not be
targeted relentlessly to vote one way or another, coerced by money and
other means?

The main question is if this is really necessary in order to battle the
SDAP's influence? Seems to me like this system also is able to be gamed by
those willing to spend the time to gather the followers. I think I have a
better system for this problem, I will share once these questions are
addressed, just to make sure I have it right.

Cheers
