> One difference is DemocracyLab segments content ratings into two
> pieces - Agreement and Importance.

There does seem to be allot of overlap. When thinking about scoring I
also thought it would be important to be able to distinguish, between
how important you thing a resolution is even when you don't agree
entirely with it its current form. I'm still not sure how the scoring
will be implemented and that is an aspect we will probably want to
experiment with.

> A final, significant difference is that DemocracyLab supports more distinct types
> of content.  MetaScore's Resolutions translate to Policies on DemocracyLab -

I also don't see the differentiation between content types as all to
important. Basically a resolution is just a file in the repository,
and it can be anything you want it to be, just like an article in a
wiki. Of course if it should be considered law, it has to follow a
certain format, but otherwise there are no limitations.

Organization of the content will revolve largely around tags and
links, and otherwise there are allot of concepts that seem to be
overlapping. The main difference I see at the moment, is that I would
like to give people a web interface for editing a repository, similar
to a wiki and the concept of code review, a nice example of which you
can find here:
http://www.djangobook.com/en/1.0/chapter01/

Each "comment" will actually be a message, to a specific part of a
file in the repository.

I'm not sure about the explicit rating of links anymore, it seems like
something people would ignore. I'd rather do that indirectly, by
registering how often a link is used, otherwise it might just end up
like using tags for messages, in that people don't see a reason to use
them.

Is there any code I can take a look at?

written proposal is of course also a good thing.
