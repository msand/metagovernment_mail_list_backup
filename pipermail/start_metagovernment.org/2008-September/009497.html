On Fri, Sep 26, 2008 at 12:36 PM, Mark Frischmuth <mark at democracylab.org> wrote:
> Thanks for pointing this out Chris.  So everyone knows, Google's App Engine
> is a distributed platform for open source software development.  Basically
> your application on their servers.  It's free, and apparently is extremely
> easy to work with.  Currently, it only supports applications written in
> Python.  Google also offers a Web Toolkit with libraries you can use to
> build your applications.  Far from a sinister force superimposing its
> malicious will, Google is actively supporting open source development.
> DemocracyLab has chosen to build its application on the App Engine.  It
> provides superior infrastructure at zero cost.

I wonder how hard it would be to get access to the source code for
"Moderator" - from playing around a little with the Google Application
Engine, the requirement for a Google Account seems arbitrary (all you
get from the user class after authentication is a 'nickname' and email
address), so it should be easily augmented with OpenID or any other
authentication mechanism.

I second the ease-of-use argument.

With regards the concern of Google holding the data - it's not a
concern with regards control:
1) A few lines of python is all it takes to export all the datastores
2) The SDK comes with a fully functioning web-server which implements
locally the server-side magic of GAE. It's not currently recommended
for external usage - it's not been security hardened - but if Google
ever pulled the plug the community would very quickly be able to
supply a replacement.

With this in mind the risks of using GAE seem pretty minimal.


> Alternatively, there's been some conversation about
> pooling resources, which could be an idea worth exploring.

Do you have any thoughts with regards what license pooled resources
should go under?

CCZero seems the most flexible and hassle free, and has one minor
political/awareness advantage that there would be some free exposure:
http://wiki.creativecommons.org/CC0_Implementations

CC-BY and CC-BY-SA seem the most likely to gain consensus simply due
to their popularity, although I haven't seen any arguments against the
assertion that the attribution and integrity clauses don't buy us
anything. E.g. if Microsoft launch MSGovernmentLive.Net, based off our
pooled resources without voluntary attribution - that's the basis of a
few indignant slashdot/mother jones/etc stories right there - seems
like free advertising ;-)
