>At DemocracyLab, do you ever have circumstances where something
>doesn't quite fit into a single category or concept? Or where new
>information requires it to be moved into another category (e.g. moving
>from opinion to evidence)?

We haven't launched publicly yet, so we haven't encountered this problem.
The Values >> Positions >> Policies structure will definitely necessitate
some user education.  Each Concept (Value, Position, Policy) and Comment
(Evidence, Opinion) can be rated by a user to reflect the Importance they
attach to it and their level of Agreement.  Our presumption is that if a
post is miscategorized it will receive low ratings and drift rightfully into
the data abyss.  So far we haven't considered circumstances where a post
would start as one data type, then become another.

An earlier conception of the data structure thought of the Values >>
Positions >> Policies structure more like real numbers than integers, where
users categorized posts along a continuum of specificity.  We steered away
from this, believing it would be too complicated for users to easily
understand.
