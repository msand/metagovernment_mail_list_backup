Parts of it are, parts of it are not...so the question of 'is it 
portable to non-Google servers' is actually a question of 'how much 
effort/time/money' would it take to do so?  What I mean by this is that 
the app engine uses python (which is portable/implemented on multiple 
environments), and it uses several libraries...e.g. parts of Django, a 
webapp framework, and a db framework, etc.  Several of these frameworks 
(e.g. db) is unique to Google's app engine (currently).  So if another 
framework was to be used (e.g. Django's db framework) it would mean some 
amount of work to change it over.  In the case of Django's db I don't 
think the work to do so is huge, but I wouldn't call it trivial either.
