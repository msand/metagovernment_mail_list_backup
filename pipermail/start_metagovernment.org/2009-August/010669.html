@Matias, Thanks for that link.

@Thomas and others,

I've been reading up about Metagovernment on the wikipage. Please correct me
if I'm wrong, but my understanding is that Metagovernment members are
interested in opening up governance in every way through using internet
technologies, specifically borrowing from open-source software development
practices.

While I see that as an important area of innovation, with positive
implications for direct democracy efforts, I don't think at GlobalDemo.org
we could reduce all the global democracy efforts to this level.

The definition of Metagovernment at this
page<http://en.wikipedia.org/wiki/Open_source_governance>is
particularly restrictive: "Open governance project aimed at replacing
all
governments with scored websites."

I'm really keen on seeing how we can collaborate (GlobalDemo and
Metagovernment) but I'm realising that we're both working on different
things (open-sourcing governance at all levels vs. democratising global
power by any means).

I'll give you a concrete example. Some of us in the UK are working on a
forthcoming project to get 1500 Africans voting in the 2010 general election
through 'donated' votes. While this is clearly a project that would fit
under the umbrella of globalising democracy, I don't think it would fit
under Metagovernment, as there is only a very minor internet/open-source
element. Does that make sense?

I do see lots of possiblities for collaboration with the WPE (www.tgde.org),
which is obviously aimed at direct/representative democracy via the
internet, but not sure about GlobalDemo.org

Would that be fair?

Cheers,

J
