> Still I don't know how to publish it.
> I know I can't just "publish" the files, as they include details like the
> password name for the DB, and similar sensitive data.
>
> How do people usually solve this?

DB passwords and usernames etc. should be stored in a separate
configuration file from which you can read in the values. I suggest
you look at "parse_ini_file()"
http://us3.php.net/manual/en/function.parse-ini-file.php. You then add
that file to your ".hgignore".

Set up a repo at http://bitbucket.org and add the following lines to .hg/hgrc

[paths]
default = http://bitbucket.org/pietro/vilfredo (or whatever the url is
that you choose)

Then " $ hg push " to upload your changes.

btw. you can use other names in [paths] for example

chris = http://bitbucket.org/chris/vilfredo

and then with " $ hg pull chris " you can pull the most recent version
Chris has uploaded.


I'm sure the procedure is similar for git, but I haven't used it very much.
