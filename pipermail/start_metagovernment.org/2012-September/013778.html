Thanks for keeping on top of it, Michael. I have been buried in several things recently.

I added this to LocalSettings:


## 2012-09-24 ED ADDED as per
## http://metagovernment.org/pipermail/start_metagovernment.org/2012-September/004990.html
require_once( "{$IP}/extensions/TitleBlacklist/TitleBlacklist.php" );
$wgTitleBlacklistSources = array(
    array(
         'type' => TBLSRC_LOCALPAGE,
         'src'  => 'MediaWiki:Titleblacklist',
    ),
    array(
         'type' => TBLSRC_URL,
         'src'  => 'http://meta.wikimedia.org/w/index.php?title=Title_blacklist&action=raw',
    ),
);


I left in the external URL since it seemed harmless. I haven't had time to figure out the regex for the local blacklist file at
http://www.metagovernment.org/wiki/MediaWiki:Titleblacklist
Please feel free. :)

But I did add in the content from:
http://www.mediawiki.org/wiki/MediaWiki:Titleblacklist


I haven't added Extension:ConfirmAccount. We already have a lot of barriers to participation, and I'd like to avoid adding yet another. Plus adding it would give us a big approval log to deal with. If we block usernames ending in a number, I think we'll cut out the vast majority, no?
