I don't see much inclination to define the Common User Information. Maybe
OpenID is enough, because active projects are so different between them that
the only things shared are username and password.

OpenID can also manage: mail address, real name, nickname, gender, birthday,
personal website, postal code, photo, country, language and time zone.

(One minimal thing I don't like about OpenID is having to write the password
in the provider's website (It's just uncomfortable.); I understand that this
avoids website owners to use my ID on other OpenID supported sites. For the
beginning and testing, I wouldn't mind if, for example,  Pietro steals my
Vilfredo account and uses it in Votorola.)
