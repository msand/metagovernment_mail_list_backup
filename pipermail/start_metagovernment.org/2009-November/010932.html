Yes, the application looks great, nice design!

   1. When a comment is submitted, the comment's windows should close.
   2. After voting, I would like to have the possibility of changing my
   opinion.
   3. Revision numbers don't mean anything for me. They don't aid for the
   interface simpleness.
   4. I would rename "settings" article by "Candiwi settings".
   5. "Current Proposal : 72-hour review time" has its colon spaced from
   "Proposal".
   6. "Time Remaining: 22:46:26.465137" is an ugly time format.
   7. There is a text field "<proposal> is in Editing/Voting status" and
   then another, "Current proposal: <proposal>". They can be merged into one:
   "Current proposal <proposal> is in Editing/Voting status."
   8. Create article/proposal links could be converted into buttons (same
   color as tabs), so it's easier to see them at first glance.
   9. Tabs are somewhat confusing. It takes a while to get used to them.
   Maybe candiwi still looks minimalist adding 3 more tabs in order to solve
   that, this way: a new orange bar is added below the blue one (with
   infomation) that in turn is under the orange tab's bar: when a proposal is
   selected. In this bar, tabs are upside down as if they came from the blue
   bar; think in a 180 degrees rotation of the original tab bar. A tab says
   "Proposal articles", other "Proposal history", and the last "Selected
   proposal".

That's all for now. I do like candiwi.
