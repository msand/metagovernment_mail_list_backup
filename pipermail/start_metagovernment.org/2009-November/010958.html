Pietro,

I know that Google Code is really cute, snappy, friendly, etc... but...
I have just seen that you set Vilfredo's license to GPLv3 and not Affero
GPLv3. Google does not allow to set Affero GPLv3 license because it
threatens all their web-based software.

Sorry if someone here already know the differences about the two
licenses, but I think it is important for all of us working of free
software for the web to use the right license, otherwise our efforts can
backfire.

Affero GPLv3 was created by the FSF to fix a "bug" on the GPLv2. This
bug was inherited by the GPLv3 because there was no legal way of
removing it without making gpl2 and 3 incompatible. This bug is the fact
that people using the software via web (in fact via any remote
connection - including mobile) were not considered "users", and
therefore didn't have the right to download the source code (and in fact
you can't download gmail's source code, even though it is based on
GPL'ed software). It is called the "ASP Loophole"
(http://en.wikipedia.org/wiki/ASP_loophole). The fix avoids proprietary
business from providing ASP services based on Vilfredo (maybe even
improving its source code) and not releasing the modified version as it
would happen in normal GPL-licensed products.

I would suggest the use of Sourceforge.
I know, it is less "user-friendly" and more "coder-friendly"... but
trust me, we shouldn't sacrifice our software's freedom.
