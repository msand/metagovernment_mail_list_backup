Hi Ed,

thanks for the reply.

Great point..., but then it would not apply in this case.

I mean in this case the decision IS meaningful. Based on this decision we
will code Vilfredo one way or another. And this will then have an effect on
all the people who want to use Vilfredo in future.
Even if just to take the code and fork it into another project.

When I asked the question on Vilfredo I was just going to code a simple way
where each user had a fixed length, but (as you might remember) it was just
me and you supporting that solution.
The community has, de facto, decided that they did not want any hard limit.
Right now I think it is very improbable that we solve the wall of text in
the hard limit way.
So already the discussion has had some effect. And the community that is
being affected is the community of all the people utilising Vilfredo. A
community that is forming, a community that is mostly composed by
metagovernment people, but not only.

But you are right that so far Vilfredo has had only "consulting" power.
I could override its decision, and would only risk becoming unpopular.
But then if I start to override the decisions too often someone can just
shortcut me,
fork the project and code what has been decided. Vilfredo community will
then probably migrate.

A different scenario than if someone just forked the project and then had to
start building the community from scratch.
So in effect the question is not "just" consultative.

Maybe the problem is that there are too many generations. We feel that since
there is potentially no end, we can just vote next time.

Personally I am considering adding some form of karma where you gain points
each generation you vote, but you lose points every time a generation passes
in a question where you have voted in the past, and now you are not voting
anymore. This is because the community has adapted, and considered your
desires as expressed in generation x. But now you moved away making it more
difficult for Vilfredo to converge.

> I think this speaks to a general problem among all governance software
> projects: how to get people coming back such that the process has enough
> participation and thought to be meaningful.
>
> My feeling is that the only sure answer here is that the software has to be
> meaningful in itself. That is, it has to have real-world effects.
>
> What is really needed is for an outside group to adopt one of our projects
> (or a combination of them) as their actual governance mechanism.
>
> The only problem is that none of our projects is ready for that. Even White
> House 2, with its superb interface and thousands of users, has no internal
> mechanism for acting as an actual decision-maker. (Nor can it anyway, since
> its thousands of users come from it having a US-wide scope: a realm where it
> is not likely to become the dominant decision-making mechanism for quite a
> while.)
>
> What projects need now is to be raised to a point of readiness where they
> can be adopted by very small organizations willing to be governed by
> software which admittedly will still be in beta.
