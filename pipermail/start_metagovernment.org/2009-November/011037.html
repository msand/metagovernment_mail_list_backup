Patrick wrote :
> If the sender's email addresses is not guaranteed (are they spoofable?),

There is a way to encrypt/sign an email, and this process is improving
which makes it less intrusive to the users ...

It would be an interesting project to encode a website more or less
entirely via email ... only, instead of passing params by URL you pass
them by subject .. putting GET in the subject, POST in the body ?

You could perhaps write a plugin for each of the (admittedly many)
email clients, that would make this easier.

On this note, I would add a related suggestion :

If security is not a primary concern, one might also consider as a
simple solution, just putting a link in an email -- an HTML link with
params & a signature.

So it could say (in the email)

Response 1 : This is a test response
Do you approve of this response ? click here:
http://vilfredo.org/email/?question=1&response=1&key=[some_encrypted_string_here]

Where each key is unique to a person, and could be used to identify
that person -- at least, within reason.

Or, if users have logged in previously and their browser still has
their cookie, they could just click the link to register their
selection
