I've contemplated the pro and con list idea before, I came to a few
problems that you might be able to address. 1) You would need a neutral
category as well, if a user has a question about the proposal, this is
neither pro nor con. 2) Do you intend to have a pro con list to each
comment or just the proposal? I think this would gum up any flow of
conversation once a debate began. 3) People gaming the system, say adding a
pro to the con list instead and getting it 5 star rated so it stays at the
top. Makes for additional moderating/flagging that isn't necessary
otherwise.


Having every comment listed on one feed is easier visually I think. The
comment header should be enough to determine if this comment is a pro or
con statement.

If these re-drafts were fair enough to gain the support of the original
proposer as well as all the highest rated commenters then there would be no
issue, right? Everyone agreed this was an accurate summary. I think this is
achievable, the re-drafters include the top comments in the summary and
gain their approval. Original proposers want to see their proposal go
through to the vote so they would approve as many redrafts as they thought
was fair in hopes that top commenters would do so as well.


It's only time consuming for the people already invested in the proposal,
they should not mind one final step in selecting the best re-draft. For the
casual user it minimizes excessive text, allowing them to understand the
proposal easier, instead of combing though what could be very technical
talk.


I don't think it's venerable to exploit, the re-drafters need to be true to
