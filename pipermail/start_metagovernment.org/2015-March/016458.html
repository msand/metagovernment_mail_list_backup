> Our current direction involves keeping the data models and interfaces 
> from our first OpenDemocracy.fi prototype. We currently have a Swagger 
> specification defined. Using this, anyone can generate the boilerplate 
> code needed to create servers and clients using any of the programming 
> languages which have swagger implementations available. This creates 
> REST-ful clients and servers which allow creating categorized 
> propositions with options, casting range-votes and comments for each 
> option, claiming to be an expert in some category, trusting some 
> self-proclaimed expert in some category.
>
> If a user decides to give positive trust to some self proclaimed 
> experts in some category, then we use the votes of those experts as an 
> approximation of the opinion of the user, for this category. This 
> applies until the user decides to cast a vote in a proposition, at 
> which point we use their latest vote as an approximation of their 
> current opinion. Thus if a user feels improperly represented by any 
> collection of experts and combination of trust to them, then they can 
> just vote directly in all the topics of relevance to them. This allows 
> adapting to good experts gone bad, by removing trust or even 
> recommending others not to trust, represented as negative trust.
>
> This system might allow studying changes in opinion over time, as it 
> allows continued updating of timestamped opinion approximations, and 
> can allow monitoring the progress and perceived successes/failures of 
> implemented propositions.
>
> We're now considering to implement a minimal responsive mobile first 
> version using Meteor and Polymer, releasing this on github and writing 
> some tutorials to help others try to build similar web standards based 
> things. Optimally we could try to interface with some other projects 
> and try to properly integrate open data-sharing protocols. Hopefully 
> this could lead to fast enough iteration and gamification, such that 
> people would enjoy using the systems and improve in the process.
>
> So count us in on the presentations and discussions relating to common
