To start gathering a specification of what the software should look
like and guidelines for the project, I would like to propose mercurial
as a version control system.

Some resources:
http://www.selenic.com/mercurial/
http://www.selenic.com/mercurial/wiki/index.cgi/Tutorial
http://hgbook.red-bean.com/hgbook.html

Advantages include
 + no squabbles over who gets commit access, everybody is an even peer
 + easy setup (no need for one master server)
 + easy to use (compared to others)
 + local history (no need for connection to server)
 + good performance
 + cross platform
Disadvantages
 - tool support isn't quite as good as e.g. subversion
 - requires some rethinking if you're used to revision control with a
central repository
