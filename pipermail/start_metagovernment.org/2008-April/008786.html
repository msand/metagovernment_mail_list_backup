I've set up an initial repository at home. I don't want to give the
impression to be eager to start development of any software. The main
objective at the moment is of course gaining consensus about our
goals, and a common repository with documents we can refer to, which
describe these objectives, will be helpful.

I also don't want to force a particular VCS upon anybody, it's just
not very hard to set up, so I went ahead with it. This is just so you
can see for yourself, how easy/hard the software is to use and get a
better impression.

http://mbarkhau.dyndns.org/

You can clone the repository with "hg clone
http://mbarkhau.dyndns.org/" and later update via "hg pull". From
there you have you're own version of the repository to work in and
commit to.

Sharing you're changes is a little more complicated, and depends on
the development model we decide to use.
You can either submit patches, produced with the "hg export" command,
which anybody may decide to apply to their repository, or you can
publish your repository with "hg serve" and announce the url or ip
address of you're repository. Anybody who likes your changes, can then
merge them into their own branch.
