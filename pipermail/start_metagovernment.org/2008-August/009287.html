Thanks for the welcome! For the past 18 months I've been working
independently on an idea which tackles the problem from a lower level
- first constructing an ontological language so that the principles,
voting mechanisms, etc of a group (say Metagovernment) could be
implemented within the same system and still be talking about 'the
same things' as another group with different principles (say
SuperGovernmentPlus). Kind of like an API for political philosophies.
The ability to talk 'about the same things' and compare means that
even people with different philosophies could easily find areas of
consensus.

That said, over a hundred pages of scribbled notes and all I have to
actually show is a silly demo where I'm trying out different UI layout
concepts with nonsensical premises: (
http://thedeviousplot.com:7358/t3/ )

In the meantime, what attracted me to Metagovernment is that many of
the principles outlined are ones I think will be a critical minimum
for a successful community. In particular - synthesis scoring is
genius.

Which is a long way of saying that if I fail on the generic API front,
then I'll certainly join the metascore list and lend a hand.
