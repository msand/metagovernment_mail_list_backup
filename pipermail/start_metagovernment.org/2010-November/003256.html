At the meeting, it was suggested we try giving another go with a  
bulletin board. We have debated this over the history of this project,  
but it does seem like something that might get more participation in  
the project.

I will attempt to set up a board as a test case, and we can  
collaboratively configure it until we think it is ready for a launch  
(if we do in fact decide to launch it). No commitment for now.

The main issue at the moment is which board to try. Any suggestions?

Here is what was posted to the chat pad before we got cut off:
*  IP.Board :
     * It supports sections and subsections
     * it supports rating
     * Possibility to receive an email when a thread is updated
     * example: http://www.thetaobums.com/index.php?/forum/1-taoist-discussion/
     * http://www.invisionpower.com/products/board/
     * Is not open source

* NavFrames
     * http://www.mediawiki.org/wiki/Extension:AWC%27s_Forum
     * Allows use of a standalone board within MediaWiki context (and  
with MW auth)

* Liquid threads
     * http://www.mediawiki.org/wiki/Extension:LiquidThreads
     * Provides a way to have structured conversations on a mediawiki
     * Under active development
     * Example: http://liquidthreads.labs.wikimedia.org/wiki/Talk:LiquidThreads_testing
     * Concerns:
        * Training users on the wiki to use the solution
        * Interplay with the mailing list (which one do I use?)
        * Long term maintenance
        * Seems very primitive compared to real discussion boards  
(like?) (Like vBulletin, for example)

* Something from http://php.opensourcecms.com/scripts/show.php?catid=5&category=Forums



Additionally, we identified these aspects that a new board should have:
* Sections:
     * Section dedicated to specific objectives; with ability for  
individuals to specify their commitment to contribute
     * Open-ended section for general discussion
     * Section for each project (with subsections as necessry)

* Possibility of rating posts (for their usefulness and/or for gauging  
consensus)

* Possibilities to add tags (although I -Pietro- don't think there is  
any software with a good tag system implemented :-( )

* possibility  for people to subscribe to particular threads (getting  
a message when a  new post is written), to the whole list  via RSS or  
email
