The second is more tricky, and in a sense this is what we are trying to
solve here. I would say that workable solutions can be Vilfredo,
Sociocracy, and any system where any one can raise an issue that the
majority need to tackle to move on. They tend to work well in small groups,
but not to scale. And of course if we are dealing with moderating a single
person that person must be excluded from being able to force (i.e. voting)
an issue (albeit they can still raise it - proposing).

Vilfredo has changed a lot in those years. The new Vilfredo (Vilfredo 2) is
very different from the previous version (Vilfredo goes to Athens). Now
people vote on a continuous scale of a triangle. The aim is that if at any
moment you want to order the proposals you can. So the question can be
interrupted at any time. Thus avoiding the "agreement by exhaustion" effect
we use to have on Vilfredo.

One of the things we are experimenting on in Vilfredo, but it is still not
in the run version, is the "blind moderation" (*Ed* pay attention here).
The idea is that we still need to have someone deciding when to move on a
decision, and where to define some threasholds. But the moderator should be
neutral and not part of the discussion. And should just decide based upon
how people vote. In fact the moderator should not even be allowed to read
the proposals. But only see the patterns with which people vote. Eventually
an AI should do this work egregiously.
