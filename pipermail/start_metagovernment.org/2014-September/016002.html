Dear Steve,
thanks for the feedback.

The ability to see the difference between two proposal is FUNDAMENTAL. And
we still don't have a diff included mostly because I have not yet found a
diff that works on complex edit (you know, the ones that have bold, and
italic). If you could find one that would be great.

Respect the option to mutate instead of having new proposals: this has two
things. You already can click on a proposal and have an edit box where you
write a new proposal with the edit box pre-filled with the old proposal
elements. When you do this you also can read the comments the old proposal
received. So this sounds a lot like what you were asking, so I suspect you
never learned how to do that, which would be a UI problem.

A different request would be to change a proposal after it has already been
voted. So the mutated version would take the place of the old one. Now this
simply cannot be accepted because it goes against one of the fundamental
principle in Vilfredo which makes sure that if you support a proposal, that
will not change (so you end up supporting something different).

So, if you could help us with a diff that would be great. And the mutation
part sure.

Cheers,
Pietro
