>>         I think the difference is the context: The main problem with
>>         representative government isn't representation per se, it's
>>         that the
>>         type of people who *want* to be representatives are generally
>>         inherently unfit to do so (because almost all of them are
>>         SDAPs).  So
>>         we don't need to eliminate representation from our
>>         repertoire, we just
>>         need to find a way to ensure that those representatives are
>>         chosen by
>>         true merit and representativeness rather than because they
>>         crave power
>>         and know how to game the system to get it.
>>
>>         > Proxies make people more susceptible to
>>         > corruption, they become the obvious target for bribery if
>>         they have a large
>>         > following. I see this as dangerous and think that
>>         individuals should always
>>         > be held accountable for their own votes, never passing the
>>         responsibility
>>         > off to someone else. While this might lesson the overall
>>         votes cast it
>>         > ensures that the votes are truly theirs.
>>
>>         But by imposing this burden you're right back at the SDAP
>>         problem:
>>         Most people are just too busy or are uninterested in the
>>         details to
>>         cast an informed vote on most issues, so you end up with
>>         government by
>>         the self-selected few.  I actually don't have any data to
>>         back up my
>>         claim that this self-selected sample is highly correlated
>>         with SDAP
>>         characteristics (e.g., no one's done a study I can find on
>>         personality
>>         characteristics of voters vs. non-voters), but I do know that
>>         it is
>>         correlated with income and that income is highly correlated
>>         with SDAP
>>         characteristics.
>>
>>         Don't know if you've read the "Matchism" upgrade of "The
>>         Plan", but it
>>         includes a couple of additional features that address this. 
>>         The first
>>         is a two-phase voting process, where the closing date for
>>         proxies is
>>         the day before the final tally and everyone who's designated
>>         a proxy
>>         gets an email that says "here's the way you're going to vote
>>         unless
>>         you go in and manually change it".  This seems to me solve
>>         the "stand
>>         behind your vote" legitimacy problem, and has the additional
>>         benefit
>>         that that email will also contain a running tally of the
>>         votes up to
>>         that point so that these semi-passive voters will get a clue
>>         that it
>>         is a tight decision and that it's probably worth their time
>>         to do a
>>         little more research to make an informed vote. The second
>>         features is
>>         a weighted list of proxies, which will distribute the load
>>         and so make
>>         it harder to game the system by bribing/influencing a few key
>>         people.
>>
>>         I'm currently working on the design for selecting proxies. 
>>         Certainly
>>         people will be able to pick them directly from posts they've
>>         made or
>>         by comparing a potential proxy's voting record with their
>>         own, but I'm
>>         also considering running a mini-Big Five personality test and
>>         matching
>>         based on that.  I'd use something like the survey on
>>         http://www.politicalcompass.org/ (which is interesting and I
>>         highly
>>         recommend exploring that site), but don't want someone
>>         picking a proxy
>>         using a rule like "he's a conservative and I'm a conservative
>>         too"
>>         because "representativeness" is a little more nuanced than that.
>>
>>         The current version is up now at matchism.org
>>         <http://matchism.org>, and the mailing lists
>>         are even (finally) working and I invite people to join them.  But
>>         unfortunately I'm going back to Taiwan on Wednesday for 3
>>         weeks and
>>         won't be able to fully participate until I get back...

>>         > results. Then the public choose which plan they like best
>>         by prioritizing
>>         > again for each goal.

>>         Right: I'm not exactly sure how the winnowing-down process

>>         decision.

>>         > solutions to the same problem are presented and gain equal

>>         trying things.
