I am very fond of Mailman since it is so ubiquitous. It sounds like we may be better off seeking out a different host than either Tux or our current one.

In the past, I had done a bit of research looking for hosting companies that would be willing to donate a hosting plan to us. We could try that route again. I would be happy to just open up the account I have now to multiple admins, but it is tied in with my financial particulars, so not really a bright idea. If we had a donated hosting service, then we wouldn't have that issue.

I had thought Tux would be a great workaround, but they seem very proprietary in their toolset (case in point: all of the below).



On Dec 10, 2012, at 5:37 PM, Michael Allan wrote:

> I opened a mailing list on the TuxFamily site, just for testing.
> List particulars:
> 
>  * Posting address: metagov-test at lists.tuxfamily.org
> 
>  * Request address: metagov-test-request at lists.tuxfamily.org
>    Subscribe by sending message with subject "subscribe".
>    Get help by sending message with subject "help".
> 
>  * Web archive: (not yet opened at time of posting)
>    http://listengine.tuxfamily.org/lists.tuxfamily.org/metagov-test/
> 
> General information about TuxFamily mailing lists:
> http://faq.tuxfamily.org/MailingLists/En
> 
> Possible problems:
> 
>  * Admin intensive.  There are no moderation flags, e.g. to prevent
>    new subscribers sending spam.  Either all new subscriptions must
>    be vetted individually, or all posts moderated (but see next).
> 
>  * Admin interface is primitive, especially for moderation.  There's
>    no web interface, it's all done by command messages.  There's no
>    notification of pending messages.  You must request a list, which
>    arrives by mail.  Then you send a "moderate accept" command for
>    each message to accept.  (I had to send a each "accept" command in
>    a separate message, one for each pending message, because the bulk
>    command facility did not work as advertised.)
> 
>  * Web archive is MonArcH, not Pipermail.  For example:
>    http://listengine.tuxfamily.org/lists.tuxfamily.org/slitaz/
> 
>  * No list information page.  We'd have to post our own.  The archive
>    won't back-link to it, but the signatures of the archived posts
>    will.  E.g. like the signature here:
>    http://listengine.tuxfamily.org/lists.plugfr.org/plug-discuss/2012/12/msg00001.html
> 
>  * No downloadable archives, e.g. compressed by month.
> 
>  * No subscriptions by web, just by email command.
> 
> It would be a step down from Mailman.  Possible alternatives:
> 
>  * Convince TuxFamily to offer a Mailman service.
> 
>  * Find another host for the list, but use TuxFamily for the wiki.
>    http://www.metagovernment.org/wiki/Upgrade_backlog#TuxFamily_migration
> 
>  * Find another host for both the list and the wiki.
> 
> What should we do?
