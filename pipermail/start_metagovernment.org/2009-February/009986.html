Hello everyone,
 
I'm pleased to report that DemocracyLab has deployed its first working beta
(http://democracylab2.appspot.com/).  The specific purpose of the software
is for use with Oregon 150 <http://www.oregon150.org/projects/youth-legacy/>
's Project 2059, designed to enable high school students to create a
collaborative vision for Oregon's future.
 
The software has the essential functionality we're targeting - allowing
users to rate values, positions, and policies, and to create connections
between relevant concepts.  It's missing a great deal of functionality that
will be necessary to deal with an abundance of information, once that
problem occurs.  Our development team emphasized simplicity and usability
when creating the software, believing that once we attract users we'll be
better able to determine next steps for development.
 
I'm working on a list of development priorities for our next (rapid)
iteration, and would appreciate any help or input this group can provide.
We've included a "feedback" button throughout the site, which allows for
easy page-specific suggestions.  Any comments you'd like to share with this
group are welcome here.
 
Thanks in advance for your feedback and assistance,
 
Mark
