The Proposal essentially just represents the transition of the changes made
in a user repository to community repository. In essence you are proposing
that the changes you have made to own repository are applied to the
community repository. Also Metascore works quite differently from a
wikipedia style system because every user has their own copy of the
repository and we are using a "distributed version control" system
(mercurial) to enable this. We thought it would be best to this design
metascore in this way as it is a proven system for large scale projects. The
way the Linux kernel is developed has been our main source of inspiration,
in fact it is even possible import the Linux kernel into metascore and make
changes to it :). I would recommend that before requesting changes to the
core system of metascore to make sure you have a clear understanding of how
it works. I realise that the sources online to the topic of "distributed
version control" are often badly explained and very technical, even the
wikipedia article is mediocre at best and the current interface of metascore
is not helping to much either. I would be willing to give a
presentation/screen cast on how metascore works to help clarify.
