Apart from the question of anonymity, I want to continue to push for  
my model of distributed administration. I expound on it as the last  
part of this thread in terms of protection of authenticated  
identities, but more generally, I think it is the best way to protect  
a governance system from corruption by administrators.

On Sep 10, 2010, at 10:44 AM, Richard Franks wrote:

> On Mon, Aug 9, 2010 at 10:16 PM, Ed Pastore <epastore at metagovernment.org 
> > wrote:
>> The first is like the SlashDot approach: simply allowing completely
>> anonymous input, but giving it significantly less weight. This  
>> demotes those
>> users from the level of trusted participant, but allows good  
>> anonymous
>> contributions to be elevated to as much prominence as the community  
>> thinks
>> they deserve. Of course, this applies better to discussion than to  
>> voting.
>
> Could it apply to voting, at all?

It could in a mixed discussion and voting system. Though prominent  
systems in this project seem to be leaning away from that model.

>> The second is the trust metric. There are numerous ways trust can be
>> calculated, but it all comes out to the same basic principle...  
>> allow users
>> to create non-identifying personas, and then have those personas  
>> prove
>> themselves within the community. Any persona which has gained the  
>> trust of
>> others in the community gets proportionally greater power within the
>> community. Yes, one person could develop several trusted personas,  
>> but
>> that's not necessarily corruption: each of those personas has  
>> proven itself
>> within the community (and can be downgraded by the community at any  
>> time).
>> Someone trying to set up a botnet of numerous, inter-trusting  
>> personas would
>> not be able to prove themselves outside of their own sphere, and  
>> thus all of
>> their input would collapse to that of one participant.
>
> I would attack it like this -- I'd analyse existing groups of a
> similar size to the one I wished to spoof, work out a plausible ratio
> of internal and external validations and instruct my group of
> co-conspirators to have their alternate personas follow that pattern.

That might work... but it just as likely (or probably more likely)  
would subject all co-conspirators to being downgraded to spammer status.

> I guess it's not all-or-nothing though.  You could just as easily
> integrate that solution into one where some individuals opt-out of
> anonymity and provide a real-life validation point for those who trust
> them and wish to remain anonymous themselves.

Yes, a mix of identified and anonymous provides a nice cross-check  
(among other seeming benefits).

>> The third is my system of distributed administration. In terms of  
>> anonymous
>> voting, it would mean that users would have to authenticate to the  
>> back-end
>> system, but then could present an anonymous persona to the front- 
>> end. This
>> seems open to administrator corruption, which is where distributed
>> administration comes in. Any software would have any number  
>> instances, each
>> of which would have independent administrators. Each instance would  
>> checksum
>> all the others, such that any deviation from the neutral standard  
>> (the
>> community of administrators) would be removed from the  
>> administration trunk.
>
> Couldn't a rogue or otherwise compromised administrator reveal
> everyones identity at any point?

I don't think so. As a close analogy: in modern network administration  
systems, a user's password is never knowable by the administrators.  
The password is encrypted with software-internal logic, and an admin  
cannot ever reveal it. The admin can change it and take over the  
account, but it would be immediately evident to the user that the  
password had been changed.

Likewise, an anonymous persona associated with an authenticated real  
identity could be protected by internal logic. The admin might be able  
to find out all the authenticated real identities, but (in a well- 
constructed system) should not be able to match them to particular  
anonymous personas.
