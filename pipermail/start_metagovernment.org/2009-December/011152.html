I have heard form several people now that different rules for
different articles are desirable but am still not convinced that this
is a good idea for several reasons.

I think this has more to do with needless bureaucracy then a useful feature.
The difference between an important article and another is purely
semantic this cannot be solved programaticaly, which means that users
must individually define different rule sets and where they apply.
Creating a simple interface for this where a user can easily define
rules and understand which rules apply is next to impossible. It is
hard enough for newcomers to understand the parallel work flow as it
is.

I think we should reconsider:
a) Is their really a problem that needs this.
b) What is the problem and is their a better solution.
