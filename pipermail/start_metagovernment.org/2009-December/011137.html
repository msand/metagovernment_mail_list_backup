I tentatively favor this idea. It only makes sense to use one of our  
own tools instead of another, if the tool in question can do the job.  
Since Candiwi is a wiki, it should be able to replace the existing  
wiki. (Stellar logic there.:)

My question is: what would happen with all the other content on the  
wiki? Would Candiwi be a complete replacement, or just a replacement  
of the homepage? I deliberately set up MediaWiki with the /wiki/  
directory just so that other tools could coexist on the same website.  
But I imagine it would be better to only use one wiki. In that case,  
would we have to transcribe each page over and then vote on it?

I also think Candiwi and Vilfredo work well together... along with  
other tools. I really would like to see a return to the discussion of  
standardization with open minds. We don't have to commit to anything  
yet, but we can at least be actively seeking compatibilities.

In particular, I don't think Vilfredo is the right tool for the  
question posed below. My understanding is that Vilfredo is least  
effective on yes/no questions, and more useful for open-ended  
questions where nobody yet knows the answer. (The Walls of Text dialog  
is turning out really well, for example.)

Wouldn't Votorola be a better tool for a yes/no question like this?  
Especially since Votorola likes to outsource its nonessential  
features, this seems like a good opportunity to explore  
compatibilities there.
