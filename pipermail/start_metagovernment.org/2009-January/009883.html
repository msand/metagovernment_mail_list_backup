Although I understand the prototype is just that, and not a final  
product, I want to comment early on the community structure mechanism.

As I understand it, the current prototype can support multiple  
communities. Does this make sense? In the eventual product, I was  
thinking more that each community would run its own instance of  
Metascore. However, I would like to see user accounts be highly  
portable between communities.

Might it be possible to continue with the multiple-community  
structure, but try to allow for communities to splinter or merge from  
instance to instance?

Also, should communities be so rigidly defined? Might they not be  
defined simply through creative use of tagging? Though it is a little  
difficult to envision, it seems like it would produce a very organic  
community structure.
