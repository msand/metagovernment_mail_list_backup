On Jan 4, 2009, at 12:31 PM, Michael Allan <mike at zelea.com> wrote:

>
>> I also feel it is slightly biased. I did not check out all other
>> systems, but it is hard to believe that "Votorola is currently the
>> only system that supports voter authentication."
>
> It's hard to do.  The system must enforce a single vote per person -
> no sock puppets.

It is difficult to compare Metascore here since it is on early Alpha,  
but we have discussed several authentication mechanisms.

However, if we use recursive user scoring (admittedly still in  
dispute), then no authentication is needed at all. In a reputocratic  
system, sock puppets are allowable, as they can only work if they can  
build their own reputation.

>> Also, you did not really substantiate the claim that "Votorola is
>> the only system in which a consensus is guaranteed to be
>> unforced". I would think that a simple majority vote preeceded by a
>> public debate, let's say in a form of an online forum, satisties
>> your criteria for the ideal unforced consensus.
>
> It's usually not so simple.  It depends on the issue to be decided.
> The requirement is that all options be on the table during the vote.
> Usually the options are many.  If the issue is a law, for example, the
> options are infinite - a law may take *any* form.
>
> When the options are many, it's unlikely that a single round of voting
> will yield a consensus.  Multiple rounds are usually required.  In
> fact, even with multiple rounds, there is no guarantee that people
> will *ever* reach agreement...
>
> So the system must often encourage consensus, in some way.  But this
> is difficult.  In most systems, the "encouragement" amounts to force.
> Typically the freedom of the voter is progressively restricted in
> subsequent rounds of voting.  But then the consensus is forced (like
> driving cattle into a pen).
>
> As far as I know, Votorola is the only system that doesn't employ
> force.  The voters are free to vote for all options, and that freedom
> is never taken away.

Metagovernment does not force consensus at all. One of our basic  
principles is "without consensus, there is no law." We prefer that a  
proposal die (or be synthesized) if it cannot garner consensus. This  
allows us to have no limit on the number of proposals, with the  
expectation that most proposals will fail.
