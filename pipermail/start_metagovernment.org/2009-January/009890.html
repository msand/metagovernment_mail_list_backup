I think supporting both multiple and single communities instances makes the
most sens. The amount of code needed to do this is minimal and needing to
setup a single instance is quite a barrier for a single community. I can
easily imagine the communities of a district sharing an instance. Making
this an 'install option' of the instance probably the best solution.
