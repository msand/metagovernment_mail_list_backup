Interesting position, Ned.

I have two questions:

1) How would you make sure that the people participating do not forget a
question? Do you think Ed will remember this in a fortnight? Should
Vilfredo send them weekly reminders? How could this be done without it
becoming a spam abuse?

2) Obviously you understand that not every question will always have a
consensual answer on which everybody agrees. Sometimes Vilfredo's work is
successful in finding the set of best candidates representing the
participants. In Vilfredo2.0 Vilfredo will take the hint that no one has
anything to add, to take all the information it has gathered so far, and
use it to provide a final ordering. But it looks like you probably would
not like this solution. How would you rather see the questions end up? How
should Vilfredo distinguish when the community just is sleeping, and when
the search has ended?

Thanks,
Pietro
