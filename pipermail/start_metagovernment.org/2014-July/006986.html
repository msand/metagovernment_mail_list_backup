Thanks for the response Ned.

I changed the subject line, so we don't discuss on that thread which is
used for that one proposal.

>From you mail I understand you say that each question is different. And
there is no algorithmic way around human evaluation of the case by case. I
can agree with that (notice how Vilfredo does not employ any natural
language processing technique). But still there is a need to create a
dialogue between Vilfredo and the community, so the community communicates
to Vilfredo if they wish to go on, or have the final result. And when.

I now suspect that people should be allowed to say when they would like to
see the results if no proposals appear. And then we take the median. In all
this how we administer a question is a simpler problem than how we answer
it. So once it is written, it should be possible for the community to self
administer it (even thought you did an egregious job).

And it IS necessary to send reminders from time to time. Vilfredo
participation really changed when we set that up correctly. The problem is
how and how much. But some are inevitable. Some questions don't need it,
but most questions do.

Cheers,
Pietro
