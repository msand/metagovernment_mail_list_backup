*@all*
Programming is the future literacy. It's like reading/writing.
Currently most people in the world are analphabets - but who knows who to
program takes part in the discourse of how the future will be designed,
because it will run on code and code will dictate what works and what not.
Everything will be automated and "Artificial Intelligence" will do more and
more things - what will happen exactly is decided by the many programmers
and the way they use open source libraries or write new code.

The government structure of the future will be digital and even though
everyone can talk to the programmer they trust in - it will be decided by
people who are literate, because those who are not dont really understand
whats possible nor what are the implications of certain decisions in code.

I would just publish everything that comes out of metagovernment
discussions or came out of them already on a wiki that metagovernment
people co-control.
ALL IT TAKES is some little bit of "markdown" knowledge and if we program
something or not will be based on discussions - and coding can be learned
on the side.

For example - markdown code ('it's what is used in the "readme" here =>
https://github.com/Metagovernment/metagovernment.github.io) ... it just
contains a "headline", nothing more.. here is the source code:
https://github.com/Metagovernment/metagovernment.github.io/blob/master/README.md
(*click "raw"*)


   - We can create many of those "markdown documents" and link them to each
   other and discuss what content should be contained in them.
   - we can try to apply voting and discussion techniques
   - once we agree on content, we do the update

This process can then be slowly "formalized" by AUTOMATING what we manually
agreed on works for us.
There does not need to be an administrator. It's possible to write a BOT
that is not controlled by any admin or member, but is just governed by
rules we make up.
It's called "SMART CONTRACT" and it's not too hard to do - but just it
needs to be done slowly and with a lot of thought behind. One option to
create such "decentralised independent smart contracts" is
 https://ethereum.org/dao <https://ethereum.org/dao>


It doesn't happen over night and everyone needs to understand what's going
on before we do any move - it's always 100% consensus if we decide that's
what consensus means - i think we currently have a rule that was listed in
the wiki when it still worked :-) ...which kind of defined consensus as
~80% of voters.... or something.

Part of the discussion is of course learning to code and a foundation needs
to be to publish little tutorials that teach everyone who wants to become a
citizen of a "free world" how to read & write - or in our case - how to
express themselves in code - so that they can participate

PS.: I see "github" just as a way to easily prototype solutions - and later
replace github with a decentralised alternative.
