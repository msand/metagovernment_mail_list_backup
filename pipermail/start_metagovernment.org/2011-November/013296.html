have explicitly specified that the Lacrosse ballot (the one showing 
convex and concave Borda) took several IRV rounds to resolve the winner, 
while the BCS  ballot (the one with the more clearly banded Borda) 
determined the winner immediately, without requiring a runoff. The 
cross-comparison of different systems sheds a lot of light on different 
kinds of behaviors, but I also need be more precise about which 
dimensions I am describing at any given time.

>>> What is the idea behind this talk.
>>>
>>> A lot of services out there permit to vote on proposals (or comments,
>>> or pages, ...). But it is not always clear what the vote represent. We
>>> suggest that voting should be done on a 2D triangular structure. Where
>>> people can express rapidly what how much do they understand the
>>> proposal, and how much do they agree with it.
>>>
>>> Then we discuss how doing this would give the service using this
>>> system some really powerful tool to aggregate proposals, and to
>>> automatically suggest which proposals should be rewritten (and who
>>> should rewrite them).
>>>
>>> So far the model is theoretical, but we are considering adding a
>>> simplified version on Vilfredo. Simplified because the Pareto Front
>>> does not play well with continuous valuation (it does work, it just
>>> does not filter out enough proposals). But the idea is general, and
>>> anyone else can also use it in their system.
