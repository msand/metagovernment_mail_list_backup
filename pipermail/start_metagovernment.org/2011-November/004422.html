The difference bridge shows the differences between any two positions 
in a poll. It is mostly used between voter and candidate to discuss 
changes or better wording, while being able to keep controversial 
differences. It is the raw material for the discussion and after the 
discussion has come to a better wording/concept in the draft the patch 
button allows the voter or candidate to patch single improvements into 
any of the two drafts. This makes large scale discussion with individual 
positions possible, as changes can move up and down the tree without the 
need to draft a new position. What you have seen on the bridge was first 
a difference in wording and second below a difference in formatting, 
which looked like code to you. We don't hide that atm., but we can do so 
in the future. But to keep formatting inline allows us to merge that 
with the difference patching as well, so no need yet to hide the html 
formatting "code" of the draft. This also only a problem for the 
homepage, because it is written in Html and not MediaWiki markup.
