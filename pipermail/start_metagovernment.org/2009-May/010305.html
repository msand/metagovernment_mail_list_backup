Michael, the StratML standard is still in development.  So services to
support all of the purposes outlined at
http://xml.gov/stratml/index.htm#DefinitionPurposes (as well as others which
we have not envisioned) have not yet emerged.  However, several prototypes
have been developed.  See http://xml.gov/stratml/index.htm#SearchServices 

The UID and password for Mark Logic's StratML search service prototype are,
respectively:  stratml_demo & marklogic  Among other things, it provides
advanced search capabilities.  So, for example, it is very easy to determine
that the mission statements in 30 of the 500+ plans in the StratML
collection contain the word "health" and 13 plans contain objectives named
"education".  However, it is just a prototype so, of course, it does not do
everything we might eventually expect to see in such a service.  For
example, it only indexes StratML documents posted on the xml.gov site and
does not allow others (anyone) to provide the URLs for StratML documents
posted anywhere on the Web.  Nor does it display potentially related
objective statements in a single view for comparison purposes.

I see that Joe Carmel's account has been suspended.
http://xml.gov/stratml/index.htm#Carmel  So his demo is apparently no longer
available online.  However, you can see what it looked like by checking out
slides 26 & 27 at
http://xml.gov/stratml/references/AIIMConference20090402_files/frame.htm
Screen shots of the other prototypes are also available there.

Ari Knausenberger's tag cloud demo is available at
http://xml.gov/stratml/index.htm#Ari 

The key point is that each of those service prototypes reuses the very same
StratML documents, all of whose URLs are provided at
http://xml.gov/stratml/urls.xml 

If and, hopefully, when others develop services providing more and better
features like those you are suggesting, I will be happy to post links to
them as well.
