> Manuel Barkhau wrote:
> > The biggest problem here will probably be user verification. It's hard
> > enough on one platform to ensure that one person doesn't have multiple
> > accounts, I can only imagine it will be much more difficult if we have
> > multiple platforms.

I think you guys are right, it's crucial.  Like Owen says, we can't
let the voting be gamed.

So we're talking about verification of user ID (authentication).  We
have to tie the ID of an online user (OpenID or email address) to the
identity of a real person (name, residential address, and so forth).
Right?  This would give us an online voter register, similar to the
government's offline register.

> > PGP with officially verified keys, which are passed around as user
> > credentials seems to be the only viable solution to me.

So a certifying authority (like Verisign) does the authentication?  I
think you're right, they offer that service.  But don't they charge a
fee?  It might be too high.

Another approach is to have the voters cross-authenticate each other,
using their own trust network.  Here's one implementation (half
coded):

  http://zelea.com/project/votorola/s/manual.xht#Trust

I guess other people will have ideas, too.  No problem, because if we
can agree on basic standards, then we don't have to design it all up
front.  We can afford to be open to alternatives and to experimenting
with them, side by side.
