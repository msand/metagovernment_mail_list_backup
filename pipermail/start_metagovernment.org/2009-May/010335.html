On May 17, 2009, at 8:44 PM, Kris Dev wrote:

> Ed, what is DVCS model? What is external solution to identity? How  
> would it help standardization? What is a commercial solution and how  
> would it bring undue influence?

DVCS is short for Distributed Version Control System.
http://www.southparkstudios.com/episodes/220760
I don't understand it perfectly, but here's what I know... Instead of  
one central server, it effectively allows everyone to run their own  
"server," publishing their version out to the world. Others can then  
accept or reject those distributed publications. This is how open  
source software developers develop code together. If you look at one  
of the metascore repositories, such as
https://bitbucket.org/mbarkhau/metascore/
you can see it contains bits of code from other people's repositories,  
which that developer accepted into his version.

It's a somewhat hectic system, and a little sophisticated of a concept  
to get across to the general public, but the end result is that there  
is no reliance on a privileged and no singular authority. It seems  
like a fantastic way for a forward-looking system of governance to be  
managed.

> What is external solution to identity? How would it help  
> standardization?

And external ID system would be something like OpenID, which is a sort  
of standard in itself:
http://en.wikipedia.org/wiki/OpenID
however as far as I know, it does not restrict a single person to a  
single identity.

> What is a commercial solution and how would it bring undue influence?

I just don't see it as a good thing to have a commercial entity be in  
charge of a central aspect of a democracy. If there is a profit  
motive, then there is a motive for something other than the benefit of  
the society, and there is immediately room for a conflict of interest  
there.
