this is hard to say and has to be analyzed more thouroughly.
It needs kind of a "drop-in" component (a javascript) that can be used in
*any* website, in order to add the feature of selecting anything on the
website and creating a "comment stream"....

I have some experience with creating something like that for adverts, where
spots for ads where not anywhere, but predefined and data was send back to
some server. To refine this to make it work cross browser in any kind of
environment took couple of weeks not fulltime, so to develop a nice drop-in
frontend component would probably take one month of full time work,
probably spread over a longer period instead of crammed into one actual
month.

Then there is the backend counterpart, which receives all the comments for
a website using the above mentioned frontend. Depending on the usage of
that kind of component, the backend has to serve massive amounts of
comments to all the websites using that feature to show to their users...
So this could be a service (paid with money or adverts or some other kind
of funding) ...or it could be a "self hosted thing", so server side
component could be open source, but would then require the person who wants
the "comments feature" in their page, to host the server somewhere, which
requires way more effort than simply adding a javascript tag to a website.

Its hard for me to estimate how much effort it would be to create the
server infrastructure, but i assume a single well skilled programmer could
also do that within one month of full time work, which would then be 2
months of full time work for a skilled fullstack programmer.
Personally, i don't think i could do the backend part easily, but i think i
could manage to do the frontend part.
