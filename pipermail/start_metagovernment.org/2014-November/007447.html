I was thinking a little bit more about evolutionary projects.

There are a few things that are common for most of social choice methods.
They use data from every accepted member.
They use same aggregation method.

Based on this I propose for evolutionary project to start with DATA, not 
a definition aggregation method.

Thus a start of evolutionary project is that you create your data and 
save them somewhere. You will also need to tell others that you have 
done so and where.

The rest is evolutionary. At the start some standards will probably 
emerge, later aggregation methods and pages, verification projects,...

Would you like to try it ? It would be very helpful for recognizing our 
opinions. You can also reuse the data in other projects. It is all 
decentralized......
