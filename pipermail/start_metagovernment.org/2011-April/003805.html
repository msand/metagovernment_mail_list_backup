Plan towards "integration" in the semantic web (SMW of Votorola)

Having said that I am willing to support you if adjustments or code becomes 
necessary. I don't see that though, since we use Semantic MediaWiki, which can 
export RDF like this:  
http://u.zelea.com/w/Special:ExportRDF/User:ThomasvonderElbe_GmxDe/G/p/frvp
Our "DB", Semantic MediaWiki, is much more generic than any specific format, so 
we simply have to teach it how to speak StratML (or any other format we are 
interested in). To export it we only have to transform the RDF to our desired 
output (which seems to work already for stuff like OpenDocument (which even has 
included RDF support) or PDF files. Maybe most interesting for you is the 
queryable structure of documents, so drafting within Votorola's SMW will give 
you straightforward access to create complex queries for drafts by StratML 
properties like goals, persons, stakeholders, ...

Steps:

1. Create an OWL-ontology

As you can see the formatting done by Thomas is not reflected in the structure 
of the exported RDF (non semantic content is not included), since we haven't 
defined the semantic properties which would be necessary for the structure to 
show up. Now instead of simply adding properties as we need (and therefore 
creating inconsistencies between different usage patterns like StratML and 
other formatting options), Semantic MediaWiki supports external ontologies 
which is perfect for us. All you need to do is create a valid OWL 
representation of the StratML.xsd which should be doable with a proper XSLT-
stylesheet. * No "programming" is needed. Once we have an equivalent 
representation of the StratML.xsd (which has nothing to do with Votorola in 
particular, but rather enables access to the StratML structured documents from 
the semantic web), it should be stored in an official domain like the XSD is, so 
that it can be shared from the semantic web.

* Example XSLT attached. Fetched from here: 
http://answers.semanticweb.com/questions/677/xsd-to-owl-importer

2. Setup Semantic MediaWiki

We then need to import the ontology in the semantic media wiki, which will 
allow us to structure drafts in a StratML compatible way. This is documented 
here: 
http://semantic-mediawiki.org/wiki/Help:Import_vocabulary
We would be able to draft StratML exportable documents then. This might be the 
tricky part, since some parts of StratML will overlap with the already 
existing data from Votorola. I think subproperties might help here, but I am 
currently learning MediaWiki setup and development myself, so I am not sure 
how to solve that yet. This is not mandatory though, it is only desirable to 
keep the properties unified between different ontologies. We can then use 
semantic forms to create is to fill in web forms for templates containing the 
StratML properties. Have a look here it is extremely flexible: 
http://www.mediawiki.org/wiki/Extension:Semantic_Forms

3. Transform RDF back to StratML 

Last we need an XSLT stylesheet to transform the OWL output back into StratML
Again no programming is needed since all we do is XML transformations. It 
might be tricky though to write this XSLT stylesheet. Again this stylesheet 
would not be specific to Votorola or even Semantic MediaWiki, but would simply 
map the XSD with the OWL ontology and would be therefore a candidate for 
standards approval as well. We might need to add an "Export" functionality for 
formats like that then.
