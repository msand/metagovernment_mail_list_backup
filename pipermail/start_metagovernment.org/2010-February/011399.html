On Feb 12, 2010, at 8:50 AM, Michael Allan wrote:

> Ed Pastore wrote:
>> By sorting projects into tiers, it might help clarify the realm in
>> which the free-range voting would occur. And more importantly, where
>> it would not apply.
>
> Where would it not apply?  Where would a voter's freedom of choice
> with regard to voting tools (or other e-democracy facilities) have to
> be restricted to a single supplier?

Sorry, I didn't mean that some tools should be excluded entirely, but  
that they should not be grouped together. That is, if there is to be  
free-range voting, it may need to have several discreet ranges.

It has already been pointed out that some tools have entirely  
different focuses. Rather than trying to make them work in tandem,  
they would work better in sequence (e.g., ideas are developed in one  
and decided in another).

This adds additional layers of complexity to the concept of the free- 
range voting, but it also makes it seem more practical. Those layers  
are: defining the ranges (such as the ones I suggested in my earlier  
post), and then defining which tools apply to which ranges  
(understanding that some tools may apply to more than one).
