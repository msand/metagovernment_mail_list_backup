On Feb 9, 2010, at 5:24 AM, Michael Allan wrote:

> Votorola has no mechanism for acceptance or rejection.  (Neither does
> WhiteHouse2, if I recall.)  You'd need a separate technical layer in
> order to pin an ultimate decision on an issue, and seal it's fate.  If
> you wanted, you could use the extra layer provided by Candiwi for this
> purpose.  (That would be a personal choice, of course.  Others might
> not choose to follow the same rules.)

Then it seems we would benefit from viewing our projects as tiers in a  
process. Some projects might cover more than one tier, but most don't  
cover all. (The exception being Telematics Freedom Foundation, which  
has a set of interacting tools for just this purpose.)

By tiers, I mean something like:

- Issue identification
- Proposal development
- Issue debate
- Proposal voting
- Proposal enactment (decision-making)

These tiers might be better defined by someone else... this is off the  
top of my head.

By sorting projects into tiers, it might help clarify the realm in  
which the free-range voting would occur. And more importantly, where  
it would not apply.
