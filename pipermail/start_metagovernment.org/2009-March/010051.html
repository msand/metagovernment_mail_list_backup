Mark Frischmuth wrote:
> I heard about stratml from Jim Gilliam here at Transparency Camp.  It sounds
> like it has some similarities to what you're proposing:
> http://xml.gov/stratml/index.htm. 

StratML is more ambitious.  It's for collaborative planning.  Projects
could use it, optionally - it's orthogonal.

GlassTop is brain-dead simple.  It's purely a transparency mechanism,
showing which stakeholders (projects) are currently supporting which
standards, and which standards are supported by stakeholders.  It does
nothing else.  (Nothing else is needed, at this point.  Especially not
between competitors.)

  My thinking now is, there won't be any custom software at the user
  end.  The user will have an off-shelf query tool - user's choice.
  There may be some scripting at the back end, where the indeces on the
  central Wiki are converted to RDF, readable by the tool (or we just
  use a semantic Wiki, for that).

  http://groups.google.com/group/changecamp/msg/110e9ac225826371

So it's easy to code.  A nice little project for a rainy day.
