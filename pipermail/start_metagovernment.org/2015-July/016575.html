Not only that, but imagine:

   1. I create a website featuring my proposal of some kind
   2. Some people just "forks" the page on github (or any other git hosting
   service) and changes the proposal a bit
   3. We discuss our DIFFs and eventually create PULL REQUESTS to merge
   reconciliation proposals to MERGE everything back into "one proposal"

The interesting thing for me is, that EVERYONE has a copy (eventually a
modified copy) of the proposal, so no singe authority can manipulate the
proposal without others recognizing it.

*FURTHER MORE:*
On Github it is possible to see ALL FORKS that have been made of a specific
repository and you can query those forks via API, which means, I can
display INFORMATION on my proposal page which depends on what OTHERS
display on their proposal pages. This gives a lot of interesting
opportunities.

   1. Auto-discover PROPOSAL SUPPORTERS => People who forked my proposal
   repository without any changes to it
   2. Display colors in the text of my proposal that indicate which lines
   divert the most/least from other proposals

...there are probably more opportunities to realize :-)
But it needs that kind of basic mechanisms at its foundation to be able to
come up with those kind of solutions in the first place.
