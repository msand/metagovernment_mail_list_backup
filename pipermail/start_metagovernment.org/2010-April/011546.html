I had an idea called decompilation-friendly language. I'm not a student of
computer sciences and I'm not sure if this idea is good or already in
practice. Most of decompilations are for reverse engineering and not for
authoring a software, I guess. I would suggest a project for creating a high
level vote-computing language (normal people should be able to read it) that
can be taken from computer language to binary, and from binary to computer
laguage again without loss of... I don't know how to call it, but the
decompiled code must always be identical to the original source code, not
just equivalent.
