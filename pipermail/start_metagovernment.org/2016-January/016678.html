Hi Ed,

I was already suggesting an alternative to the wiki, which would be to move
to GITHUB :-)
I created a copy of the current page for now and the url is
http://metagovernment.github.io which forwards to http://metagovernment.org
What is missing is a forwarding from http://metagovernment.org to
http://metagovernment.github.io
I can do it, but it needs access to the domain name registrar - a
description how to do it can be found here:

   - https://pages.github.com/
   -
   https://help.github.com/articles/setting-up-a-custom-domain-with-github-pages/

*Why is that important?*
There is a language called "markdown" (see and try here:
http://jrmoran.com/playground/markdown-live-editor/ )
In that way, our WIKI would or could become an entry *HOME.md* file which
could include a "Table of Content", followed by lots of other *.md *files,
which are linked from *HOME.md*
or other *.md *files. That way, the whole wiki would be just a collection
of markdown files that you write according to that editor link.
Those files will be all stores in a files & folders structure you can see
here: https://github.com/Metagovernment/metagovernment.github.io

What you see in that last link is the current "web page" copy of
http://metagovernment.org

Whenever something on the wiki / web page should be changed, it can be
discussed in "Github Issues", see here:

   - https://github.com/Metagovernment/metagovernment.github.io/issues


The web page is not the only thing the "Metagovernment organization" has on
github:

   - https://github.com/Metagovernment

There is also the "*ACB*" and the "*Community*" Repository

   - https://github.com/Metagovernment/ABC  was opened by *Mlchal *(which
   seems to be an abandoned user account) so i would suggest to delete it if
   nobody likes ABC)
   - The other one is https://github.com/Metagovernment/Community were some
   discuss already, see here:
   https://github.com/Metagovernment/Community/issues

*PLEASE JOIN THAT LAST LINK's DISCUSSIONS and OPEN YOUR OWN ONES :-)*

With all of the above comes:

   - an associated chat that can be used with a github user account:
   https://gitter.im/Metagovernment
   - Each Metagovernment Repository can have it's own chat and it will show
   notifications around the associated "repository" (e.g. changes to the home
   page or wiki files and folders)
   - an associated project management board, which is JUST a different view
   on "github issues", for example see:
   https://waffle.io/metagovernment/Community

In addition i already tried to request access to the mailinglist with
another metagovernment-bot email address, so i could forward all
notifications or activity around the metagovernment github organization to
this mailinglist :-)

... that way we get something that's well maintainable and everyone can
contribute, while in the wiki that was barely the case
