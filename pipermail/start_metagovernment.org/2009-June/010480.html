In the spirit of full disclosure, I have to admit that I'm having a hard
time with the MetaScore User Flow.

>From the prototype it seems like a discussion board with a hand full of tabs
that don't quite make sense, although I get the feeling that the discussions
should somehow result in a final document. To this end, I went about
imagining how a discussion could somehow be captured in a document. I've
sketched a few of the ideas out on paper, but I wanted to share my thoughts
before scanning it in.

To begin with: A Document is merely a collection of Articles. No matter how
complicated things might get, remember that a Document is merely a
collection of Articles. :)

Articles are divided into Sections. An Article may contain innumerable
Sections. Each Section is defined by its Paragraphs, of which it may contain
many. (Ie. Article 1, Section 2, Paragraph 2 of the United States
Contsitution states...) Each of these: the Article, the Section, and the
Paragraph, are part of the Header. As such, they may be voted or commented
upon, or recommended anew by those with the appropriate permissions
(described below). New Articles, Sections, Paragraphs, and Comments may also
contain Tags and References (described below).

Underneath the Header is the Footer, which contains the Document's metadata.
This metadata includes the Authors of the Document, and its permissions,
along with a list of Tags and References. Tags are simply words or phrases,
separated by a comma, that describe the document. Tags are displayed in a
Tag Cloud, with the most frequent tags appearing larger than those with
single instances. References are links to websites which somehow relate to
the Document. The permissions and terms of use may only be set by the
creator, who may invite others to contribute, or open the document for
community involvement.

On top of the document is a series of Tabs which select its point of view.
The first tab is the View Tab which displays a view of the Document. The
second tab, the Discuss Tab, displays the Discussions on the Document,
sorted chronologically, with a link to their originating Section, Article,
or Paragraph. The third tab, the Priority Tab, displays a list of proposed
modifications to the Document, sorted by recent activity, with a link to its
origin as well. The fourth tab is the Code Tab and contains a link to the
Source Code of the Document. The fifth and final tab is the History Tab, and
displays a chronological list of all the changes to and clones of the
document.

To the right of the Document are it's tools that change according to which
tab is selected. The View Tab displays an optional image at the top of the
tools, above the Document's table of contents. The tools for the Discuss Tab
contain details about the conversations currently taking place, with a
button that invites the user to Start a Discussion. The Priority Tab
displays a set of tools that enable the user to create new Priorities. The
Code Tab shows helpful information on how to copy and edit the Document,
while the History Tab includes detailed information about the activities
related to the Document.

And on top of it all rests the Document Bar -- a thin little strip that
begins with the Application Title: Metascore. To the right of MetaScore is
the Document Name. On the far right of the Bar is the User Name, a Settings
link, and a final link to Log Off. The Application Title, MetaScore, leads
to a list of all the Documents currently open on MetaScore. The tools to the
right display a log in prompt, or a button to create a New Document (should
the user be logged in). Clicking on the Document Name resets the Document to
its initial state. The User Name leads to a Recent List of Activity by the
User, while the Settings link to their private Settings.

I'll leave the final Log Off link to the imagination.

Let me know if I'm anywhere close and I can revise the process accordingly.
