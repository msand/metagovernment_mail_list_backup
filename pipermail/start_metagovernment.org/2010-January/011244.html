Hello,

Pietro Speroni di Fenizio schrieb:
> I still have doubts. For example in Vilfredo you don't vote for things
> once and for all,
> you vote on them, and revote, and revote. And each time you refine.
> Sometimes the same user votes for something in one generation, and
> then ignores it in another generation. And all this is part of the
> process that sends implicit feedback to the other users.
> I don't see how in this system it would work to have it all mirrored. 

Yes, as far as I understand, Vilfredo is very unique among the other 
voting-tools. But I think, that some of the information users give to 
Vilfredo can be mirrored into other tools. At the very least the last 
vote each user casts in a certain question before a new generation has 
started. If he doesnt participate in the new generation then its an open 
question, wether he gave up to convince the others or was convinced 
himself by the majority of the others.

If I understand Vilfredo right, this is actually an interesting question 
for every user. Does the result reflect the majority now or is it just a 
measure of the patience of the users to convince the others, i.e. their 
felt importance of the debate?
Maybe this question could be answered by the users, if you add an option 
like: "I still stay with my last vote, but dont participate any longer."

In that case even the votes from former generations could be exported.

> Still I see more problems coming out. For example, the pareto front
> works perfectly well both if you use binary voting (support/ignore) a
> three votes (YES, maybe, NO), n-values ranking, complete ranking, and
> so on. In fact it is such a general concept that, provided that in
> each generation a user only votes using one system, Vilfredo could
> cope with that too.

Mirroring votes is just a way of translating. Each mirroring engine has 
to decide, how he tanslates something. This can be done well or not well 
(then it will be critiqued for the bad job). If you want to translate a 
ranked voting (-10 to +10) into a binary, information will be lost. But 
it is still a valid information to tanslate the +10 votes into a "Yes". 
I would even say all votes from +1 to +10 are translatable to "Yes".

> What would you do about the rooms? The room existence in private. you
> might be aware of some rooms, but in general the rooms are there to
> let people have a place to discuss things in private. If I were to let
> an external entity know about what rooms are active, the whole privacy
> issue would be gone.

One way could be to only allow public rooms to mirror. Another could be, 
that in a private poll the mirroring engine also only allows the same 
users to participate.
