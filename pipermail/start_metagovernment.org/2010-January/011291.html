I've been using the wiki a lot, lately.  I have two requests:

CSS
---
It's hard to layout tables properly.  Could a sysop please paste the
following CSS into this page:
http://metagovernment.org/wiki/MediaWiki:Common.css

--- quote ------------------------------


  table.wikitable
  {
      border-collapse: collapse;
      margin-top: 1em; margin-bottom: 1em;
  }


  table.wikitable td, table.wikitable th
  {
      border: 3px solid #DDD;
      padding: 0 0.05em 0.05em 0.3em; /* TRBL */
      vertical-align: baseline;
  }


  table.wikitable td
  {
      background-color: #F9F9F9;
  }


  table.wikitable td.clear
  {
      background-color: inherit;
      border: none;
  }


  table.wikitable th
  {
      background-color: #F2F2F2;
      text-align: left;
  }

------------------------------ end quote

Usage:

  {| class='wikitable'
   |-
   ! header 1
   ! header 2
   ! header 3
   |-
   | row 1 cell 1
   | row 1 cell 2
   | row 1 cell 3
   |-
   | row 2 cell 1
   | row 2 cell 2
   | row 2 cell 3
   |}

Captcha
-------
I'm presented with a captcha on every edit that adds an external URL.
I guess you're using the defaults, defined here:
http://www.mediawiki.org/wiki/Extension:ConfirmEdit

Those are probably too restrictive.  Ordinarily, you don't need
captchas for registered users:
http://www.mediawiki.org/wiki/Extension:ReCAPTCHA#Configuration

This all goes in your LocalSettings.php.  Here's a working example
with typical settings (see bottom of page):
http://zelea.com/system/host/t/var/www/localhost/htdocs/mediawiki-c/LocalSettings.php

If registered users are spamming a lot, maybe do this:

  $wgGroupPermissions['user'         ]['skipcaptcha'] = false;
  $wgGroupPermissions['autoconfirmed']['skipcaptcha'] = true;

Then users who automconfirmed their email addresses aren't captcha'd.
